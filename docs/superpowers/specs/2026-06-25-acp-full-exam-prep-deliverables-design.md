# 设计：阿里云大模型ACP考试备考三件套交付物

## 概述

基于《阿里云大模型ACP考试大纲-V3》与《大模型ACP认证教程》(C1-C5) 全部章节，产出三份独立、可发布的备考交付物：知识点梳理手册、考点清单、模拟考试卷（含答案与解析）。三份文档自洽互链，覆盖官方 6 大模块 100% 核心考点，模拟卷严格贴合官方真实试卷结构。

本设计为 H 级。生成采用**方案1：中央顺序生成**——先定全卷答案分布蓝图，再按模块逐块命题，中央统一去重与质量自检。不使用并行子Agent（会削弱答案分布与场景一致性的中央控制）。

## 范围

### 构建
1. `exam_materials/common/ACP知识点梳理手册.md`
2. `exam_materials/common/ACP考点清单.md`
3. `exam_materials/common/ACP模拟考试卷_含答案解析.md`

### 不构建（out-of-scope）
- 不含「案例分析题」独立题型（官方真实试卷只有单选+多选；用户已确认严格贴合官方）。应用/方案/排错能力通过**场景化单选/多选题**考察。
- 不改动现有 `common/六大模块模拟卷`、`C2/C3/C4` 分章题库（仅作质量参考与去重依据）。
- 不生成扩展题库（用户确认只出 1 套 75 题标准卷）。
- 不含交互式网页/打印组件，仅 Markdown。

## 官方事实基线（来自考纲V3）

- 试卷满分 100，及格 80，时长 120 分钟。
- 单选 50 题（各 1 分）+ 多选 25 题（各 2 分）= 75 题 / 100 分。
- 模块占比：应用开发 17% / 提示词 15% / 检索增强 20% / 微调 16% / 多Agent多模态 16% / 生产实践 16%。

## 架构：教材章节 → 官方模块映射

| 官方模块 | 占比 | 主要教材来源 |
|---|---|---|
| 大模型应用开发 | 17% | C2_1（API/参数/消息/流式批量）；C1 环境 |
| 大模型提示词工程 | 15% | C2_3（提示词框架/CoT/Few-shot/系统角色/Meta Prompting/上下文工程） |
| 大模型检索增强 | 20% | C2_2（知识范围）、C2_5（RAG优化：句子窗口/自动合并/解析/重排）、C2_4（RAGAS评测） |
| 大模型微调 | 16% | C4_1（蒸馏与微调：作用/前提/算法/数据集/参数/评测） |
| 多Agent及多模态 | 16% | C3 全章（工具调用/规划执行/多Agent协作/Memory/Skill/评测/Qwen Code）+ 百炼Assistant多模态 |
| 生产环境应用实践 | 16% | C4_2（vLLM/ECS/FC/PAI/百炼部署）、C4_3（生产实践）、C4_4（内容安全合规/云上安全） |

## 组件

### 文档1：知识点梳理手册
按 6 模块组织。每模块含：
- 教材章节映射 + 模块概述
- **知识点图谱表**：列 = 知识点 / 对应教材位置 / 考察权重(高·中·低) / 考察形式(识记·理解·应用) / 常见易错点
  - 权重：源自模块占比 + 考纲强调度（高=核心高频，中=常规，低=了解性）
  - 形式：源自考纲动词（"了解"→识记，"理解"→理解，"掌握/动手/部署"→应用）
- 模块小结：高频考点提示

### 文档2：考点清单
从手册提炼高频核心考点。清单式，每考点一行/一块：
- 考点名 · 所属模块 · 考察维度(识记/理解/应用) · 易错方向 · ★高频标记 · 对应模拟卷题号
- 末尾「考点→题号覆盖矩阵」，证明模拟卷 100% 覆盖核心考点。

### 文档3：模拟考试卷
- 第一部分 单选 50 题，第二部分 多选 25 题，按上表模块配比分布。
- 每题标注 `【模块·考点·难度(基础/中档/拔高)】`，题干场景化（操作/方案设计/故障排查导向）。
- 卷末：标准答案表 + 逐题考点解析（考点 + 正确项理由 + **每个干扰项为何错**）+ **答案分布自检表**。

## 数据流：答案分布蓝图（破解"答案扎堆"红线）

命题前**先固定答案位**，命题时填入：

- 单选 50 题正确项配额：A=13, B=12, C=13, D=12（每模块内部也尽量打散，不连续同字母）。
- 多选 25 题正确项**组合规模**全类型覆盖：2 选 9 题、3 选 11 题、4 选 5 题；且正确项字母组合分散（不全为 ABC）。
- 解析末尾用自检表统计实际分布，必须与蓝图一致方为通过。

```
考纲动词/占比 ──┐
教材内容 ───────┼──▶ [手册:知识点+易错点库] ──▶ [清单:核心考点] ──┐
                │                                  易错点库 ─────────┤
答案分布蓝图 ───┴────────────────────────────────▶ [模拟卷:题干+强干扰项+解析]
                                                          ▲
                                          覆盖矩阵校验 ────┘（清单考点 100% 命中题号）
```

## 错误处理 / 质量门（生成后自检，全过才算完成）

| 检查项 | 阈值 |
|---|---|
| 题量结构 | 单选=50、多选=25、模块配比=上表 |
| 单选答案分布 | A/B/C/D ≈ 13/12/13/12，无某字母 <10 或 >15 |
| 多选组合覆盖 | 2/3/4 选均出现，无"全为ABC" |
| 干扰项强度 | 每个干扰项对应一个具体认知误区/易混概念，解析能说清「为何错」 |
| 考点覆盖 | 清单每个核心考点 ≥1 题命中（覆盖矩阵无空行） |
| 场景导向 | 中档/拔高题为操作/方案/排错场景，非纯名词背诵 |
| 占位符 | 0 个 TBD/TODO |

## 测试策略
- happy path：三份文档生成，质量门全过，覆盖矩阵满覆盖。
- error path：分布偏离蓝图 → 调整对应题正确项重排；覆盖矩阵有空行 → 补题。
- edge case：教材未直接覆盖但考纲提及项（如 vLLM 部署、FC 快速发布、云上安全要素）→ 以考纲为准命题并在手册标注来源。

## 关键决策
1. **不设案例分析题**（用户确认严格贴合官方）：应用能力靠场景化单选/多选承载。
2. **中央顺序生成而非并行子Agent**：保证答案分布与场景一致性中央可控；并行非用户要求。
3. **先定答案分布蓝图再命题**：从结构上消除"答案扎堆"红线。
4. **干扰项由手册易错点库驱动**：保证强迷惑性，非临场编造。
5. **三份独立新文件**（用户确认），放 `exam_materials/common/`，现有材料仅作参考与去重。

## 未知项
无显式延期项。考纲"可能包括未列明内容"的部分，以教材实际内容兜底，不额外外推。
