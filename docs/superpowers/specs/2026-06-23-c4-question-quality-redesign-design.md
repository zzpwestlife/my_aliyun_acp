# C4 题目质量重做设计

## 背景

当前 `exam_materials/C4/模拟题题目.md` 与 `模拟题答案解析.md` 已经完成一轮扩题，但用户实测后给出两类非常明确的反馈：

- 多选答案个数和组合过于机械，容易形成“默认猜三个答案”的做题策略
- 干扰项偏弱，很多错误选项可以只靠常识或语气判断快速排除

这说明 `C4` 的主要问题已经不是题量不足，而是命题质量不足。现有题库虽然覆盖了蒸馏、部署、生产治理和安全合规四大主题，但不少题目停留在概念识别层，缺乏真实工程场景中的约束竞争、优先级判断和上线治理判断，因此区分度偏低。

本次设计目标不是重新定义 `C4` 的知识边界，而是在保留当前题号范围、章节结构和主干知识点覆盖的前提下，对 `C4` 做一轮中等力度的题库重做，使其从“概念覆盖卷”提升为“需要理解工程边界与系统治理才能稳定做对”的高质量客观题卷。

## 范围

本次只处理以下两个文件：

- `exam_materials/C4/模拟题题目.md`
- `exam_materials/C4/模拟题答案解析.md`

本次不处理：

- `exam_materials/C4/考试大纲.md`
- `C2`、`C3` 的任何文件
- `common/` 下的综合资料
- 新增 PDF、打印稿或其他导出物

## 核心决策

### 1. 采用“中修重做”而不是只修分布或整套推翻

本轮采用中修方案：

- 保留现有题号 `1-38`
- 保留当前知识点主干覆盖
- 系统重写大部分干扰项
- 对泄题明显或判断路径过直的题目，重写题干
- 同步重写对应解析

不采用“只修分布”的原因：

- `C4` 的问题不止是答案分布机械
- 很多题目的错项本身就过弱，只打散答案字母和组合并不能真正提升区分度

不采用“大修”的原因：

- 用户本轮目标是继续重做 `C4`，但没有要求整套推翻重建
- 大修会显著扩大改动面，增加解析重写成本和回归风险

### 2. 保留题号与结构，重做工程判断质量

本轮不改 `C4` 的整体组织方式：

- 单选题 `24`
- 多选题 `14`
- 总题数 `38`

保留以下稳定约束：

- 题号不变
- 章节结构不变
- 提交答案区与统计区结构不变

重做重点放在：

- 题干中的业务与工程约束
- 干扰项质量
- 单选与多选答案分布
- 解析中的错项分析质量

### 3. 用“同维度竞争”替代“绝对化错误项”

所有重写后的题目都应遵守一个核心命题原则：

- 错误选项不能错得过于明显
- 错误选项必须与正确选项处于同一决策维度
- 错误选项应错在前提、适用边界、工程代价、治理深度或上线顺序

重点减少以下低质量错项模式：

- “只要……就一定……”
- “完全不需要……”
- “天然等价……”
- 与题干场景明显离题的凑数项

## 设计目标

### 1. 提升真实考试感与工程感

重做后的 `C4` 不应再只是“认概念”，而应更像真实考试：

- 给定预算、时延、流量、运维能力等约束，判断更合适的技术路线
- 比较两个都合理的方案，选更优先或更完整的一个
- 判断某种蒸馏、部署、缓存、安全或上线治理策略的边界

### 2. 打散答案分布，消除模式泄题

本轮明确把“答案分布自然化”作为设计目标之一。

单选题要求：

- 不允许大量集中在同一选项
- 分布可以不完全平均，但必须自然
- 连续多题答案相同必须来自题面逻辑，而不是命题习惯

多选题要求：

- 不能继续形成“几乎全是三个正确项”的规律
- 需要自然引入 `2` 项、`3` 项、`4` 项混合答案规模
- 组合变化必须由题面逻辑驱动，而不是机械打散

### 3. 提高干扰项强度

重做后的干扰项应更多来自以下几类“高质量错法”：

- 路线本身可行，但不符合当前约束
- 技术能力更强，但工程代价或治理成本不匹配
- 局部正确，但无法解决题干的核心矛盾
- 上线顺序不合理，跳过了必要的验证、灰度或审计环节

### 4. 保持 `C4` 知识点覆盖不缩水

虽然重做题面，但不能因为追求难度而丢掉原本 `C4` 应覆盖的主干能力：

- 蒸馏与微调边界
- 数据合成蒸馏 / 黑盒蒸馏 / 数据过滤
- 部署与发布选型
- API / 百炼 / vLLM / FC / PAI-EAS / ACK / ECS
- TTFT、缓存、批量推理、`max_tokens`、输出约束
- 不默认依赖大模型
- 提示词注入 / 指令注入
- AI 安全护栏、最小权限、参数校验、审计

## 文件级设计

### 一、`模拟题题目.md`

保留当前四段结构：

- 单选题
- 多选题
- 提交答案区
- 题型统计

重做方法如下：

- `1-18`：保留知识点方向，但重写明显直给的题干与弱干扰项
- `19-32`：重点打散多选答案规模与组合，并提高错项竞争性
- `33-38`：保留补充单选题定位，但把新增题拉到与前半卷一致的难度标准

优先重做的题型特征：

- 一眼能排除两个以上选项的题
- 题干没有明确业务或工程约束的题
- 正确项明显比其他选项“更完整、更稳、更像标准答案”的题
- 多选中只要理解主题就能默认猜三个答案的题

### 二、`模拟题答案解析.md`

解析稿继续保留现有结构：

- 使用说明
- 标准答案速查表
- 单选题答案解析
- 多选题答案解析
- 考点覆盖与复习建议

同步重写要求：

- 所有被改答案的题，速查表必须同步更新
- 所有被重写题面的题，解析必须同步改写
- 错项分析不能停留在“其他项错误”这种弱解释
- 要明确指出错项错在什么约束或边界

## 质量标准

本次重做完成后，应满足以下标准：

### 1. 题目不能靠常识秒杀

理想状态不是“题目很绕”，而是“读完题必须做工程判断”。因此：

- 不能再出现大面积明显离题选项
- 不能再出现只看语气就知道哪项更像标准答案的情况

### 2. 单选答案分布自然

完成后复查单选答案分布：

- 不追求机械平均
- 但必须避免显著集中在单一字母

### 3. 多选答案规模与组合不再机械重复

完成后复查多选答案：

- 不允许继续大面积集中在 `3` 个正确项
- 不允许让“默认猜三个答案”成为高成功率策略
- 需要同时出现 `2` 项、`3` 项、`4` 项的合理组合

### 4. 解析支持复盘

每道题的解析都应至少回答：

- 正确项为什么对
- 最关键的错项为什么错
- 这题考查的是哪个工程边界

## 验收标准

完成后应满足以下条件：

- `C4` 保持 `38` 题规模不变
- 题号 `1-38` 保持不变
- 单选题答案不再明显集中在少数单一选项
- 多选题答案个数不再大面积集中在 `3`
- 多选答案组合不再由少数重复模式主导
- 至少一半以上题目的错误选项具备同维度竞争性
- 被改动题目的解析全部同步更新
- 做题者不能主要依靠答案分布规律或常识排除拿高分

## 非目标

本次不做以下内容：

- 不把 `C4` 改成场景题混合卷
- 不继续扩题到更大规模
- 不修改 `C2` 或 `C3`
- 不重写 `C4/考试大纲.md`

## 风险与控制

### 风险 1：为了打散分布而强行改答案

控制方式：

- 先按题面逻辑决定答案，再检查分布
- 不能为了分布好看而牺牲题目合理性

### 风险 2：为了提高难度而把题写偏

控制方式：

- 所有重写都必须落在现有课程与考点范围内
- 难度来自边界和条件，不来自冷门知识

### 风险 3：题面改了，但解析没跟上

控制方式：

- 按“题面 -> 速查表 -> 逐题解析”顺序同步改
- 最后做题号和答案一致性核对

### 风险 4：多选题只是在字母层面打散，实质仍然单薄

控制方式：

- 不只看组合分布，还要人工检查每题是否真的存在同维度竞争
- 至少保留一轮“弱干扰项失败模式”人工复查

## 后续路径

如果本轮 `C4` 中修重做达到预期，可把同一套标准继续用于：

- `C4` 之后的整卷模拟卷抽题标准
- `C2/C3/C4` 全题库的答案分布审计
- 后续更高质量整卷模拟卷的命题规范
