# C3 题目质量升级设计

## 背景

当前 `exam_materials/C3` 已经具备三份核心资料：

- `考试大纲.md`
- `模拟题题目.md`
- `模拟题答案解析.md`

但整体质量仍偏向“知识点识别”和“机制名称判断”，距离高质量考试题还有明显差距，主要问题包括：

- `考试大纲.md` 更像知识梳理稿，尚未承担命题基线角色。
- `模拟题题目.md` 虽然结构完整，但不少题目仍以概念识别为主，缺少对工作流选择、协作模式、记忆策略和评测闭环的综合判断。
- `模拟题答案解析.md` 已有解析基础，但缺少系统化的错项分析与题目设计层面的回扣，尚未形成和 `C2` 一样清晰的复盘链路。

本次优化目标不是改变 `C3` 的题型路线，而是在保留“全客观题”结构的前提下，把题目质量提升到和 `C2` 同一标准层级，使 `C3` 成为 Agent 模块的高质量样板。

## 范围

本次只处理以下三个文件：

- `exam_materials/C3/考试大纲.md`
- `exam_materials/C3/模拟题题目.md`
- `exam_materials/C3/模拟题答案解析.md`

不修改 `C2`、`C4` 内容，不处理 `common/` 下的综合资料，不新增 PDF、不改目录结构。

## 核心决策

### 1. 保留全客观题结构

`C3` 本轮继续使用“单选题 + 多选题”的全客观题结构，不引入场景题。

原因如下：

- Agent 模块本身包含大量机制边界、模式选择、工作流判断、组合能力设计，天然适合用高质量客观题考查。
- 当前 `C3` 已有较完整的客观题基底，重构成本低于改为混合结构。
- 通过强化条件限制、模式比较和多能力组合题，依然可以形成足够区分度，不必依赖主观场景题。

### 2. 保持 20 题体量

继续保留现有 `20` 题规模：

- 单选题 `12` 题
- 多选题 `8` 题

但重构难度结构与命题方式，不再满足于“数量完整”，而是让每道题都承担明确的考查目标。

### 3. 以命题分层替代知识堆叠

本次将 `C3` 题库改造成与 `C2` 一致的三联动结构：

- `考试大纲.md`：负责定义“考什么、考多深、适合怎么命题”
- `模拟题题目.md`：负责实现“用什么题目考、如何形成区分度”
- `模拟题答案解析.md`：负责解释“为什么对、为什么错、该回到哪里复习”

## 设计目标

### 1. 建立适合 C3 的难度梯度

`C3` 保持 `20` 题全客观题，但明确分为：

- 基础题 `8`
- 中档题 `8`
- 拔高题 `4`

基础题主要覆盖核心机制边界，中档题引入条件与模式比较，拔高题强调多能力组合、系统设计判断和工程闭环。

### 2. 提高题目的系统设计感

题目不再停留在“这个术语是什么”，而要更像真实考试：

- 工具调用：考结构化输出、外部执行、校验与重试机制，而不只是考定义
- 规划与执行：考固定工作流、自主规划、反思策略的适用边界
- 多 Agent 协作：考 Leader-Worker、Blackboard、handoff 等组织方式的匹配场景
- 记忆：考截断、摘要、向量召回、主动记忆管理的组合关系
- Skill：考从临时 Prompt 到可复用流程的沉淀逻辑
- 评测：考端到端评测、白盒评测、持续闭环的工程意义
- Qwen Code：考其工作流方式与 Agent 开发方法论的联系，而非零散命令记忆

### 3. 建立高质量解析链路

答案解析需要像 `C2` 一样成为复盘材料，而不是单纯的答案注释。每题至少要能回答：

- 为什么正确项成立
- 最关键的错误项为什么不成立
- 这题属于哪个知识点
- 这题映射到大纲中的哪一块能力

## 文件级设计

### 一、`考试大纲.md`

重构为命题基线稿，结构从“知识模块介绍”升级为“能力分层与命题映射”。

应至少包含以下模块：

- Agent 运行机制与工具调用
- 工作流、规划与反思
- 多 Agent 协作模式
- 记忆管理
- Skill 设计与沉淀
- 评测驱动开发
- Qwen Code 实践

每个模块都应补齐以下字段：

- 考点
- 考核层级
- 建议难度
- 常见命题方式
- 易错方向

输出目标是让 `C3` 不再只是“学了哪些内容”，而是明确“哪些内容适合出基础题、哪些适合出中档题、哪些适合出拔高题”。

### 二、`模拟题题目.md`

继续保持 `20` 题全客观题：

- 单选题 `12`
- 多选题 `8`

但命题方式需要重构：

- 基础题：覆盖 Agent 机制主干、常见定义边界和运行逻辑
- 中档题：加入条件、限制、模式比较和方法选择
- 拔高题：强调多能力组合、系统完整性和工程化闭环

具体设计要求：

- 至少一半以上题目带场景前提或约束条件
- 干扰项必须同维度竞争，不能一眼排除
- 多选题不能大量重复“全选型”正确答案
- 综合题必须让错误项“看起来也有道理”，但存在关键缺陷

### 三、`模拟题答案解析.md`

保留当前已有的“使用说明 -> 标准答案速查表 -> 逐题解析 -> 复习建议”主结构，但内容标准升级。

每题统一要求：

- 标准答案
- 解析
- 错项分析
- 知识点
- 考纲映射

对综合题还应补充：

- 为什么该能力组合完整
- 为什么其他组合不完整或不优先

## 质量标准

本次 `C3` 升级后的质量目标如下：

- 不再以术语识别题为主，而是以机制判断和设计选择题为主
- 不再大量出现明显弱干扰项
- 不再把多个概念机械堆在一起，而是围绕一个真实问题展开判断
- 解析必须能支撑复盘，而不只是对正确项做一句解释

## 验收标准

完成后应满足以下条件：

- `3.1` 到 `3.7` 的核心能力都有明确落题
- 题目内部能看出基础、中档、拔高三层梯度
- 至少一半以上题目带条件、场景或模式比较
- 单选题答案分布不过度集中
- 多选题正确答案组合不过度重复
- 每题解析都包含“标准答案 / 解析 / 错项分析 / 知识点 / 考纲映射”
- 整套题更像真实 Agent 系统考试题，而不是章节知识摘抄题

## 非目标

本次不处理以下内容：

- 不将 `C3` 改造成带主观场景题的混合卷
- 不修改 `C4`
- 不重做 `common/` 下的综合模拟卷
- 不处理打印版、网页版、PDF 版输出

## 风险与控制

### 风险 1：为了提升区分度导致题目过偏

控制方式：

- 所有拔高题都必须来源于课程边界和已整理的大纲
- 重点考机制组合与工程判断，不考冷门术语

### 风险 2：全客观题难以体现综合能力

控制方式：

- 通过“模式比较、系统设计、组合能力判断”提升题目深度
- 把综合判断下沉到高质量单选和多选，而不是简单加场景题

### 风险 3：大纲、题目、解析三份文件继续各写各的

控制方式：

- 先改大纲，再重写题目，最后同步解析
- 验收时用“考纲 -> 题目 -> 解析”的链路核对

## 后续路径

如果 `C3` 试点结果达到预期，下一步可沿同一方法继续升级：

- `exam_materials/C4/考试大纲.md`
- `exam_materials/C4/模拟题题目.md`
- `exam_materials/C4/模拟题答案解析.md`

届时会直接复用本次形成的：

- 命题分层结构
- 干扰项质量标准
- 解析字段模板
- 验收标准
