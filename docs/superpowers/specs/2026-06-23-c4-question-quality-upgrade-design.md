# C4 题目质量升级设计

## 背景

当前 `exam_materials/C4` 已经具备三份核心资料：

- `考试大纲.md`
- `模拟题题目.md`
- `模拟题答案解析.md`

相较于最初版本，`C4` 已经形成了较完整的题库框架，覆盖了 `4.1` 到 `4.4` 的主要知识范围，且题量充足。但从“高质量考试题”标准来看，仍然存在以下问题：

- `考试大纲.md` 更像知识框架总结稿，尚未承担命题基线角色。
- `模拟题题目.md` 虽然覆盖面广，但部分题目仍偏向知识点改写，工程判断感不足。
- `模拟题答案解析.md` 已经有速查表和解析骨架，但错项分析深度不稳定，尚未形成与 `C2/C3` 一样统一的高质量复盘链路。

`C4` 与 `C2/C3` 的不同之处在于，它对应考试中高权重、强工程化的综合模块，涉及模型蒸馏、部署选型、生产实践和安全合规四条主线。因此本次优化目标不是压缩题量，而是在保留现有大卷规模的前提下，提升命题分层、题目区分度和解析价值，使 `C4` 成为“交付上线”模块的高质量题库样板。

## 范围

本次只处理以下三个文件：

- `exam_materials/C4/考试大纲.md`
- `exam_materials/C4/模拟题题目.md`
- `exam_materials/C4/模拟题答案解析.md`

不修改 `C2`、`C3` 内容，不处理 `common/` 下的综合资料，不新增 PDF，不改目录结构。

## 核心决策

### 1. 保留 32 题体量，不压缩题量

本轮继续保留 `32` 题结构：

- 单选题 `18` 题
- 多选题 `14` 题

原因如下：

- `C4` 的知识跨度大，涉及 `4.1` 到 `4.4` 四个工程板块，压缩题量会影响覆盖度。
- `C4` 本身在考试中的权重高，保留大卷体量更接近真实刷题体验。
- 当前问题不在于“题量太多”，而在于部分题目虽然数量够，但质量层次不够清晰。

### 2. 保持当前难度分布，但重定义每层含义

继续采用当前整体分布：

- 基础题 `12`
- 中档题 `14`
- 拔高题 `6`

但重定义各层含义：

- 基础题：覆盖蒸馏、部署、SLO、安全的主干边界
- 中档题：引入条件、约束和工程取舍，考选型与优先级
- 拔高题：考跨模块的完整链路设计，而非冷门知识点

`C4` 的拔高题不追求偏题或怪题，而是强调“工程完整性判断”，例如蒸馏与在线生成的边界、部署与成本的权衡、安全护栏与权限模型的职责边界等。

### 3. 以“工程化判断题”替代“知识点改写题”

本次升级的核心不是换题型，而是换命题方式：

- 从“概念是否记住”转向“工程上如何判断”
- 从“服务名是否背会”转向“场景下如何选型”
- 从“安全手段名称是什么”转向“纵深防御各层分别负责什么”

最终目标是让 `C4` 的题目更像真实考试中的生产交付与上线题，而不是章节摘要题。

## 设计目标

### 1. 建立适合 C4 的命题分层

`C4` 的 `32` 题将继续分为基础、中档、拔高三层，但每层的职责更加明确：

- 基础题：帮助考生确认核心概念边界，不允许继续出现明显送分项
- 中档题：通过场景化约束考部署、评测、性能与安全的判断能力
- 拔高题：通过综合题考察“蒸馏 -> 部署 -> 上线治理 -> 安全”的整链设计思维

### 2. 提高题目的工程判断感

四个子模块的题目要分别完成升级：

- `4.1` 蒸馏与微调：从“知道蒸馏是什么”升级为“会判断哪些任务适合蒸馏、数据应该如何过滤、评测应优先看什么”
- `4.2` 部署模型：从“知道百炼 / vLLM / EAS / FC”升级为“会根据团队能力、流量形态、时延目标和运维成本做部署选型”
- `4.3` 生产实践：从“记住 TTFT、缓存、批量推理”升级为“会判断需求优先级、体验指标、稳定性设计与降本路径”
- `4.4` 安全合规：从“记风险名称”升级为“会理解提示词注入、工具风险、护栏能力边界以及权限模型的职责分工”

### 3. 建立高质量解析链路

答案解析不仅要告诉用户“哪项是对的”，还要回答：

- 为什么正确项更符合工程现实
- 最关键的错误项为什么看起来对、但本质上不对
- 这题落在哪个知识点
- 这题映射到大纲中的哪一块能力

对于拔高题，还需要额外说明：

- 为什么这是一组完整设计
- 为什么其他方案只是局部正确或优先级错误

## 文件级设计

### 一、`考试大纲.md`

重构为命题基线稿，而不是知识摘要稿。

应覆盖以下主模块：

- 蒸馏与微调边界
- 模型部署与发布选型
- 生产环境应用实践
- 安全合规与纵深防御

每个模块都应补齐以下字段：

- 考点
- 考核层级
- 建议难度
- 常见命题方式
- 易错方向

输出目标是明确：哪些内容适合出基础题，哪些适合出中档题，哪些适合出拔高综合题。

### 二、`模拟题题目.md`

保留 `32` 题结构：

- 单选题 `18`
- 多选题 `14`

但重构命题方式：

- `4.1` 重点考任务边界、数据质量、评测优先级、蒸馏适用条件
- `4.2` 重点考 API / 百炼 / vLLM / PAI-EAS / FC / ACK / ECS 的选型逻辑
- `4.3` 重点考需求分析、功能性需求与非功能性需求、TTFT、缓存、稳定性和降级
- `4.4` 重点考提示词注入、工具注入、黑名单边界、AI 护栏与权限模型

命题质量要求：

- 至少一半以上题目带条件、场景或权衡关系
- 干扰项必须同维度竞争，不能一眼排除
- 多选题答案组合不能出现明显机械重复
- 综合题必须考“完整工程链路”，不能只是多个知识点拼盘

### 三、`模拟题答案解析.md`

保留当前已经较成熟的主结构：

- 使用说明
- 标准答案速查表
- 单选题答案解析
- 多选题答案解析
- 考点覆盖与复习建议

但内容标准要统一提升。

每题统一要求：

- 标准答案
- 解析
- 错项分析
- 知识点
- 考纲映射

对拔高题还应补充：

- 为什么这是更完整的工程化设计
- 为什么其他选项只是局部成立或优先级错误

## 质量标准

本次 `C4` 升级后的质量目标如下：

- 不再以知识点改写题为主，而以工程判断题、方案选型题和系统边界题为主
- 不再出现明显弱干扰项
- 不再把多个概念机械堆叠成综合题，而是围绕一个真实生产问题展开判断
- 解析必须能支撑复盘和二次学习，而不只是给出正确答案注释

## 验收标准

完成后应满足以下条件：

- `4.1` 到 `4.4` 的核心能力都有明确落题
- 题目内部能看出基础、中档、拔高三层梯度
- 至少一半以上题目带条件、场景或权衡关系
- 多选题答案组合不过度机械重复
- 每题解析都包含“标准答案 / 解析 / 错项分析 / 知识点 / 考纲映射”
- 拔高题能体现跨模块的完整链路判断
- 整套卷更像生产交付与上线的工程考试题，而不是章节知识摘抄题

## 非目标

本次不处理以下内容：

- 不压缩 `C4` 题量
- 不将 `C4` 改造成主观题或场景问答混合卷
- 不修改 `C2`、`C3`
- 不重做 `common/` 下的综合模拟卷
- 不处理打印版、网页版或 PDF 输出

## 风险与控制

### 风险 1：为了提升工程感导致题目过长

控制方式：

- 场景只保留必要约束，不写成冗长背景故事
- 每题聚焦一个核心判断点，不把多个维度塞进同一道中档题

### 风险 2：保留 32 题后，仍然出现重复考法

控制方式：

- 先在大纲中明确每类题的命题边界
- 对多选题和综合题做答案组合与考法分布检查

### 风险 3：安全模块和部署模块继续停留在概念识别层面

控制方式：

- 强制把 `4.2` 与 `4.4` 的题目写成“方案选择”和“职责边界”判断题
- 综合题必须覆盖模型、工具、权限、评测或上线治理中的至少两个维度

## 后续路径

如果 `C4` 升级达到预期，`exam_materials` 目录下的三大模块将形成统一标准：

- `C2`：问答系统与 RAG 题库样板
- `C3`：Agent 系统题库样板
- `C4`：交付上线与生产治理题库样板

届时可再考虑对总索引文档 `README.md` 做一次统一更新，说明三套题库已经完成按同一质量标准的升级。
