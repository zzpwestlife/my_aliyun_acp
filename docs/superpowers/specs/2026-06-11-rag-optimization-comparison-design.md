# RAG 优化方法对比表设计

## 背景

当前复习已经涉及 RAG 优化相关概念，但“句子窗口检索”“自动合并检索”“重排序”“标题改写”分别解决什么问题、位于 RAG 哪一层、与 ACP 考纲和当前课内内容是什么关系，仍然容易混淆。

为了支持后续复习，需要一份以考试大纲为主视角、同时兼顾课内边界的对照文档。

## 目标

- 产出一份适合 ACP 复习的 RAG 优化方法对比表
- 以考试大纲优先的方式解释 4 个核心概念
- 明确标注“考纲视角”和“课内是否明确出现”的区别
- 提供可直接记忆和区分的结论，降低做题混淆

## 范围

本次只覆盖以下 4 个概念：

- 句子窗口检索
- 自动合并检索
- 重排序
- 标题改写

本次不扩展：

- 新的模拟题
- 新的错题分析
- RAG 全流程总览文档
- 代码实现或 Notebook 修改

## 交付物

新增一份 Markdown 文档：

- `/.trae/specs/generate-c2-mock-exam/rag-optimization-comparison.md`

## 文档结构

### 1. 标题与说明

开头说明本文件采用“考纲优先”视角，适合考试复习；同时提醒用户，部分术语可能在当前课内材料中没有同名原词。

### 2. 核心对比表

使用 Markdown 表格，对 4 个概念做横向比较。

建议比较维度：

- 概念
- 一句话定义
- 主要解决的问题
- 所处阶段
- 对 RAG 的主要作用
- 课内是否明确出现
- ACP 复习建议

### 3. 结论区

补充 3 类结论：

- 从技术作用上怎么区分这 4 个概念
- 从 ACP 考试上怎么记忆
- 从当前 C2 课内复习上如何把“课内原词”和“考纲扩展词”分开

### 4. 速记提示

最后补一小段简短速记法，帮助快速区分：

- 谁负责“召回更多对的内容”
- 谁负责“把碎片拼完整”
- 谁负责“排序更准”
- 谁负责“让块本身更容易被召回”

## 风格要求

- 面向考试复习，不写学术化长文
- 定义简洁，优先可记忆
- 术语表达清晰，不混用“课内明确出现”和“通用技术常识”
- 不引用未经当前仓库材料支持的细节实现

## 验收标准

- 文档包含 4 个指定概念
- 文档包含一张可读的 Markdown 对比表
- 文档明确区分“考纲优先”和“课内是否出现”
- 文档最后给出可直接背诵的结论
- 不改动现有题库文件
