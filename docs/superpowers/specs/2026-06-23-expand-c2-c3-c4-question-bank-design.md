# 扩充 C2 C3 C4 题库设计

## 背景

当前 `exam_materials` 目录下的 `C2`、`C3`、`C4` 三个模块，已经完成了一轮按统一质量标准的题库升级：

- `考试大纲.md` 作为命题基线
- `模拟题题目.md` 作为模块题面
- `模拟题答案解析.md` 作为复盘材料

但如果以真实考试规模作为参考，现有三套题更像“高质量第一轮精练题”，还不足以支撑更接近正式考试总量的系统化刷题。用户希望继续扩题，并明确提出以下目标：

- 只扩 `C2`、`C3`、`C4` 三个模块
- 总题量尽可能贴近甚至略高于真实考试规模
- 题目要尽可能多而全
- 题目之间不能重复，不能只是换皮改写
- 继续保持当前质量标准，不因为扩题而降低区分度和解析质量

本次设计的目标，就是在不改变当前文件组织方式的前提下，把三套模块题库扩成更接近正式考试规模的大题库。

## 范围

本次只处理以下 6 个文件：

- `exam_materials/C2/模拟题题目.md`
- `exam_materials/C2/模拟题答案解析.md`
- `exam_materials/C3/模拟题题目.md`
- `exam_materials/C3/模拟题答案解析.md`
- `exam_materials/C4/模拟题题目.md`
- `exam_materials/C4/模拟题答案解析.md`

不修改：

- `C2/C3/C4` 的 `考试大纲.md` 主体结构
- `common/` 下的综合资料
- `README.md`
- 其他模块

## 核心决策

### 1. 只扩 C2、C3、C4

本轮只扩：

- `C2`
- `C3`
- `C4`

不扩其他章节或其他模块。

原因如下：

- 当前仓库内已经明确建立了这三个模块的高质量题库基础。
- 这三个模块也是当前教程体系里最适合直接转化为高质量客观题训练材料的部分。
- 用户已经明确确认“只扩 C2/C3/C4 三个模块即可”。

### 2. 直接在现有题库文件中追加，不另起第二套卷

本轮采用追加模式，而不是拆卷模式：

- 不新增“第二套”
- 不拆成“基础卷 / 强化卷 / 冲刺卷”
- 不把现有题迁移到新文件

而是直接在现有 `模拟题题目.md` 与 `模拟题答案解析.md` 中继续向后追加。

原因如下：

- 当前每个模块已经形成固定的题目与解析格式。
- 用户明确选择“直接在现有文件里继续扩充，做成每个模块的大题库”。
- 追加方式更利于后续按题号、按模块和按知识点做连续复盘。

### 3. 保留现有题号不变，只向后续号段扩展

本轮不重新洗牌，也不重排旧题号。

规则如下：

- 旧题号全部保留
- 新题直接从当前最后一题之后继续追加
- 答案解析中的题号必须与题面完全同步

原因如下：

- 用户已明确选择“保留现有题号，后续直接追加新题”。
- 当前三套题已经建立了题号与解析的一一对应关系，重排会破坏已有复盘习惯。

## 目标题量

本轮扩题后的目标规模如下：

### C2

- 单选题扩到 `18`
- 多选题扩到 `10`

### C3

- 单选题扩到 `16`
- 多选题扩到 `10`

### C4

- 单选题扩到 `24`
- 多选题保留 `14`（现有即已达到，不删旧题）

### 总量

- 单选题合计 `58`
- 多选题合计 `34`

这个总量会略高于真实考试 `50` 单选 + `25` 多选，更适合作为分模块刷题题库，而不是单次整卷。

## 配比原则

本次扩题不平均分配，而是按考试价值与模块权重倾斜：

- `C4` 扩题最多
- `C2` 次之
- `C3` 再次之

原因如下：

- `C4` 覆盖蒸馏、部署、生产治理、安全合规，是高权重综合模块
- `C2` 覆盖应用开发、提示词、RAG 和评测，是主干基础模块
- `C3` 虽然重要，但现有题面已经具备较高密度，扩题规模可以略小于 `C2/C4`

## 模块级扩题策略

### 一、C2 扩题策略

`C2` 继续围绕以下能力扩题：

- Prompt 设计
- RAG 基础链路
- 检索优化
- 自动化评测
- 参数控制与回答质量优化

新增题重点：

- 不重复考已有的基础参数定义题
- 更多考检索链路判断、召回与重排序思路、评测指标和优化顺序
- 多选题要增加“哪些做法同时成立、哪些优化手段适用”的判断题

### 二、C3 扩题策略

`C3` 继续围绕以下能力扩题：

- 工具调用
- 工作流与规划执行
- 多 Agent 协作
- Memory
- Skill
- 评测驱动开发
- Qwen Code

新增题重点：

- 不重复已有的模式名称识别题
- 更多考协作模式选择、机制边界、反思与反馈、Skill 设计质量、评测闭环
- 保持全客观题风格，但提高系统性判断比例

### 三、C4 扩题策略

`C4` 继续围绕以下能力扩题：

- 蒸馏边界
- 模型部署与发布
- 生产环境应用实践
- 安全合规与纵深防御

新增题重点：

- 强化任务是否适合蒸馏、部署路径是否合理、上线前先做什么、安全职责分别属于哪一层
- 增加更完整的工程链路判断题
- 保持“生产交付与上线”的工程感，不回退成概念背诵题

## 质量标准

本次扩题必须遵守以下质量约束：

### 1. 不重复

不允许出现以下情况：

- 旧题原样换一种说法
- 只改一个场景词，考核点完全相同
- 同一个知识点在同一难度层重复问同一种判断

允许的扩展方式是：

- 同一知识点换一个新的工程角度
- 同一模块增加新的边界问题、组合问题或优先级问题

### 2. 不降质

新增题必须继续满足：

- 干扰项同维度竞争
- 不出现明显一眼排除项
- 中档题带约束或条件
- 拔高题体现系统完整性而不是冷门偏题

### 3. 解析继续完整

新增题的解析必须继续保留：

- 标准答案
- 解析
- 错项分析
- 知识点
- 考纲映射

不能因为题量增加而退化成“只给答案，不给分析”。

## 文件级设计

### 一、`模拟题题目.md`

每个模块的题目文件继续沿用当前结构，但追加方式按现状分别处理：

- `C2` 保留原有“单选 / 多选 / 场景题”三段，并在文件后部追加“补充单选题 / 补充多选题”
- `C3` 保留原有“单选 / 多选”两段，并在文件后部追加“补充单选题 / 补充多选题”
- `C4` 保留原有“单选 / 多选”两段，并在文件后部追加“补充单选题”
- 三个模块都统一更新提交答案区与题型统计

本次只做以下变化：

- 保留旧题原位和旧题号
- 从当前最后题号继续追加新题
- 不为追求按题型连续而调整旧题顺序
- 更新题型统计
- 更新提交答案区，使其覆盖新增题号

### 二、`模拟题答案解析.md`

每个模块的答案解析继续沿用当前结构，并允许在旧章节后增加“补充单选题答案解析 / 补充多选题答案解析”小节：

- 使用说明
- 标准答案速查表
- 原有逐题解析
- 新增补充题逐题解析
- 考点覆盖与复习建议

本次只做以下变化：

- 保留旧题解析原位
- 从当前最后题号继续追加新增题解析
- 更新速查表，覆盖全部题号
- 保留 `C2` 的场景题关键词表，不改旧场景题编号
- 更新考点覆盖与复习建议，使其反映扩题后的覆盖范围

## 验收标准

完成后应满足以下条件：

- `C2` 达到 `18` 单选 + `10` 多选
- `C3` 达到 `16` 单选 + `10` 多选
- `C4` 达到 `24` 单选 + `14` 多选
- 三个模块的题号均连续追加，不重排旧题
- 三个模块的答案解析与题面题号完全一一对应
- 新增题不存在明显重复或换皮现象
- 新增解析仍包含“标准答案 / 解析 / 错项分析 / 知识点 / 考纲映射”

## 非目标

本次不做以下内容：

- 不新增整卷模拟卷
- 不拆成基础卷、强化卷、冲刺卷
- 不扩 `C1/C5` 或其他模块
- 不重写 `README.md`
- 不重做大纲文档主体

## 风险与控制

### 风险 1：为追求总量而出现题目换皮

控制方式：

- 扩题前先按模块列出现有题已覆盖的角度
- 新题必须优先补“尚未充分覆盖”的角度

### 风险 2：题量增加后解析质量下降

控制方式：

- 坚持逐题写完整解析字段
- 优先保证错项分析质量，而不是只追求解释正确项

### 风险 3：追加后题号和速查表不同步

控制方式：

- 每个模块扩题后做题号数量校验
- 解析稿用脚本检查题号与题面一致性
