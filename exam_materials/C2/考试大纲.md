# C2 考试大纲

本文档用于支撑 `C2` 模块题库重构，不仅整理《阿里云大模型 ACP 考试大纲》中与“构造问答系统”相关的核心要求，也明确每类考点适合出的题型和难度层级，作为 `模拟题题目.md` 与 `模拟题答案解析.md` 的命题基线。

## 一、模块定位与权重

- 模块名称：`C2 构造问答系统`
- 对应教程：`2_1` 到 `2_5`
- 对应考试大纲板块：大模型应用开发、提示词工程、大模型检索增强
- 模块特征：权重高、实操性强、既考基础概念，也考 RAG 工程化判断与迭代方法

按现有教程映射，`C2` 主要覆盖以下三块内容：

| 模块 | 对应权重 | 对应教程 | 备考重点 |
| --- | --- | --- | --- |
| 大模型应用开发 | 17% | `2_1` | API 参数、消息机制、多轮对话 |
| 提示词工程 | 15% | `2_3` | 系统提示词、模板设计、任务表达 |
| 大模型检索增强 | 20% | `2_2`、`2_4`、`2_5` | RAG 基础链路、优化方法、自动化评测 |

## 三、命题难度分布建议

建议将 `C2` 题库组织为一套 `18` 题的标准样板：

- 单选题 `10` 题：基础 `4`、中档 `4`、拔高 `2`
- 多选题 `6` 题：基础 `2`、中档 `3`、拔高 `1`
- 场景题 `2` 题：诊断优化 `1`、评测决策 `1`

对应命题重点如下：

| 难度 | 目标 | 典型题型 |
| --- | --- | --- |
| 基础 | 覆盖主干考点，检查概念边界与流程掌握 | 参数作用、系统提示词、RAG 基础顺序 |
| 中档 | 引入条件、场景和方法比较，提升区分度 | 优化选型、错误定位、指标理解 |
| 拔高 | 基于课程边界做综合权衡，不考偏题怪题 | 拒识策略、优化优先级、版本决策 |

## 四、出题重点与易错方向

### 1. 基础题重点

- 参数作用不要只靠机械记忆，应能结合问答场景判断。
- 消息机制容易和系统提示词职责混淆。
- RAG 基础链路容易在“解析、切片、召回、重排序”顺序上出错。

### 2. 中档题重点

- 很多考生知道“要做 RAG 优化”，但不知道先改哪一步。
- 提示词工程常见误区是只会堆内容，不会结构化表达。
- 评测题容易把“有输出”误判为“效果更好”。

### 3. 拔高题重点

- 需要区分“模型本身答得像对”与“确实基于知识库答对”。
- 需要在提示词优化、检索优化、评测闭环之间判断先后顺序。
- 需要理解工程化迭代不是单次调参，而是基于统一样本和指标做比较。

## 五、命题约束

- 不出脱离课程边界的冷知识题。
- 不使用明显异维度的弱干扰项。
- 至少一半以上客观题应包含条件、场景或方法比较。
- 单选题答案分布不能过度集中，多选题正确组合不能大量重复同一模式。
- 场景题必须能写出标准答案与评分要点，不能只给开放式讨论题。

## 六、从大纲到题目的落地原则

为了保证 `考试大纲.md`、`模拟题题目.md`、`模拟题答案解析.md` 三者联动，本模块采用以下落地原则：

- 大纲先定义“考什么、考多深、适合出什么难度”。
- 题目再落实为“如何考、如何制造区分度、如何避免送分项”。
- 解析最后落实为“为什么对、为什么错、回到哪个知识点复习”。

这意味着 `C2` 不只是知识梳理模块，也应当成为后续升级 `C3`、`C4` 题库质量的样板。
