## 一、单选题答案解析

### 1. 【基础】某团队做企业制度问答，希望模型回答更稳定、随机性更低。以下哪项调参方向最合理？

标准答案：B

解析：对于制度问答、知识问答这类偏事实型任务，通常希望输出更稳定、更少发散，因此应适当降低 `temperature`。课程中的基本思路是把创造性让位给稳定性，而不是让模型自由发挥。

错项分析：A 会增加随机性；C 只改变返回方式，不改变答案稳定性；D 与多轮对话保留有关，但不是直接控制随机性的参数。

知识点：参数控制与生成稳定性

课程映射：`2_1_用大模型构建新人答疑机器人.ipynb`

考纲映射：大模型应用开发 -> API 参数作用

### 2. 【基础】一个答疑机器人支持连续追问，但用户一旦换种说法继续提问，模型就经常忘记前文。最关键的改进是？

标准答案：C

解析：多轮对话的关键是持续维护 `messages` 列表，让模型能看到系统角色、用户历史和必要的 assistant 回复。如果上下文链断掉，模型即使能力再强，也只能把后续追问当成新问题处理。

错项分析：A 会直接丢失上下文；B 不解决历史缺失问题；D 会把知识库和会话上下文混在一起，既浪费上下文，也不能替代对话历史维护。

知识点：消息机制与对话历史

课程映射：`2_1_用大模型构建新人答疑机器人.ipynb`

考纲映射：大模型应用开发 -> 对话管理

### 3. 【基础】在提示词工程中，系统提示词最核心的作用是？

标准答案：A

解析：系统提示词负责给模型设定相对稳定的角色、任务边界和输出约束，是整个任务的“长期规则”。它不是替代用户输入，而是给后续所有用户输入提供解释框架。

错项分析：B 夸大了系统提示词的作用；C 过于狭窄，长度控制只是副作用之一；D 明显错误，系统提示词在调用阶段就会生效。

知识点：系统角色提示词

课程映射：`2_3_优化提示词改善答疑机器人回答质量.ipynb`

考纲映射：提示词工程 -> 角色设定

### 4. 【基础】一个基础 RAG 流程中，较合理的顺序是？

标准答案：D

解析：基础 RAG 流程通常是先做文件解析，再进行文本切片，随后根据用户问题召回候选片段，最后再通过重排序筛选更相关内容。这个顺序体现了“先构建可检索单元，再做检索和筛选”的工程逻辑。

错项分析：A 在还没解析文档前就切片，流程不成立；B 和 C 都把召回提前到了切片之前，违背了 RAG 链路的前提关系。

知识点：RAG 基础链路

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`

考纲映射：大模型检索增强 -> RAG 基础构建

### 5. 【中档】某企业内部知识问答系统经常回答出“听起来合理但并不来自公司制度”的内容。若只能优先做一项改进，最应该先做哪项？

标准答案：B

解析：题干的核心问题不是“模型能力不够”，而是“答案没有被知识证据约束”。优先做法应是强化基于检索证据作答，并在证据不足时明确拒答，从而先压制模型的通识扩写倾向。

错项分析：A 成本高且未必解决幻觉来源；C 会让表达更发散；D 删除切片会破坏检索基础，通常只会让问题更严重。

知识点：基于证据回答与拒识

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> RAG 与直接作答的边界 / 拒识策略

### 6. 【中档】你要让模型把“请帮我审批明天请假的申请，我是因为发烧”这类输入分类，并稳定输出 JSON 字段 `intent`、`reason`、`urgency`。哪种提示词设计更合理？

标准答案：C

解析：这道题考的是结构化任务设计。最有效的方法是明确任务、使用分隔符组织输入，并通过固定 JSON 模板限制输出字段，让模型在“怎么做”和“输出成什么样”两个层面都得到清晰约束。

错项分析：A 把结构化工作交给后处理，稳定性差；B 只加背景不加结构，仍容易跑偏；D 流式输出与格式正确性没有直接因果关系。

知识点：提示词模板与结构化输出

课程映射：`2_3_优化提示词改善答疑机器人回答质量.ipynb`

考纲映射：提示词工程 -> 提示词要素、分隔符、模板设计

### 7. 【中档】RAG 系统已经能命中相关句子，但回答经常遗漏句子前后的限制条件，导致答案不完整。以下哪种优化最有针对性？

标准答案：A

解析：题干不是“完全召回不到”，而是“命中了句子但上下文不完整”。句子窗口检索的价值就在于在命中句附近补充前后文，适合修复“零碎命中、条件丢失”的问题。

错项分析：B 调低随机性解决不了证据不完整；C 删除系统提示词属于错误方向；D 把所有切片塞进上下文会带来噪声和截断风险，不是精准优化。

知识点：句子窗口检索

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 召回策略优化

### 8. 【中档】你要比较两版问答系统：A 为基础 RAG，B 加入标题改写、表格增强和重排序。哪种评测方案最可靠？

标准答案：D

解析：要比较两版系统的真实差异，必须把变量控制住。最可靠的方法是在同一批代表性样本上跑两版系统，同时比较回答质量和检索支撑情况，这样才能把改动效果从样本差异中分离出来。

错项分析：A 只能得到主观印象；B 会引入样本偏差；C “有输出”不等于“答得对”，不能作为有效评测标准。

知识点：版本对比与评测设计

课程映射：`2_4_自动化评测答疑机器人的表现.ipynb`

考纲映射：自动化评测 -> 评测集设计 / 同批样本比较

### 9. 【拔高】某问答系统在少量问法下已经能召回正确片段，但最终答案仍会混入模型自身通识知识。若时间有限，优先级最高的改进组合是？

标准答案：C

解析：题干已经说明“召回有时是对的”，问题主要出在生成阶段没有被证据严格约束。因此优先应做三件事：强化基于证据回答、优化重排序提升证据质量、在证据不足时拒答。这是成本和效果最平衡的组合。

错项分析：A 直接微调成本高且定位不准；B 盲目扩上下文容易带来更多噪声；D 放弃 RAG 会让知识边界彻底失控。

知识点：证据约束、重排序与拒答联动

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 拒识 / 基于证据回答 / 优化优先级

### 10. 【拔高】你只有一周时间对答疑机器人做一轮有效迭代。当前已知问题包括：部分问题召回错误、部分答案过于冗长、且团队没有统一评测口径。最合理的优先顺序是？

标准答案：A

解析：这道题考的是工程化迭代方法。没有统一评测口径时，团队无法判断“改得更好”还是“只是改了”。因此第一步应建立代表性评测集和基线，在此基础上再决定优先修检索还是提示词，才能形成可比较的迭代闭环。

错项分析：B 和 D 都属于先动手再判断，容易重复试错；C 切模型既昂贵又掩盖根因，不适合作为首要动作。

知识点：评测基线与迭代优先级

课程映射：`2_4_自动化评测答疑机器人的表现.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

## 三、场景题参考答案

### 17. 场景题：新人答疑机器人命中零碎片段

参考答案：

1. 问题诊断：当前至少存在两类问题，一类是问法变化后召回不稳，说明检索覆盖和召回鲁棒性不足；另一类是命中内容过碎，说明切片与上下文拼装策略有问题。
2. 优化顺序：先建立一组代表性问题定位问题类型；再优先检查检索链路，包括文本切片策略、查询表达、句子窗口检索或自动合并检索；然后补充“仅基于证据回答”和证据不足时拒答的提示词约束；最后用统一样本做自动化评测和人工抽检。
3. 每一步的原因：如果检索不到或只取到碎片，后续提示词再好也难以纠正；当检索质量改善后，再通过证据约束降低模型补写通识内容的倾向；最后通过评测验证优化是否真的有效，而不是只靠主观感觉。
4. 对应课程章节与考纲落点：课程主要对应 `2_2`、`2_5`、`2_4`；考纲主要对应大模型检索增强中的基础链路、召回优化、实战优化和自动化评测。

解析：题干中的两个症状都指向“检索链路优先于生成链路”这一判断，因此最合理的顺序不是直接改模型参数，而是先修召回和上下文，再补生成约束，最后用评测闭环收口。

知识点：RAG 问题诊断、召回优化、证据约束、评测闭环

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_4_自动化评测答疑机器人的表现.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 基础链路 / 召回优化 / 实战优化 / 自动化评测

### 18. 场景题：如何用自动化评测指导迭代

参考答案：

1. 评什么：至少评回答准确性、回答是否真正被检索证据支撑，以及检索结果本身的相关性和完整性。
2. 怎么比：准备一组覆盖典型业务问题和难点问法的代表性样本，让版本 A 和版本 B 在同一批样本上运行；保留统一输入、统一评价口径，并结合自动化指标和人工抽检。
3. 如何根据结果决定下一步：如果版本 B 的检索支撑明显更好但答案仍不稳定，优先继续优化提示词和输出约束；如果检索结果本身仍不相关，优先回到切片、标题改写、表格增强和重排序策略继续调优；如果两者都明显提升，再进入下一轮更精细的业务用例扩展。
4. 对应课程章节与考纲落点：课程主要对应 `2_4` 和 `2_5`；考纲主要对应自动化评测、版本比较和评测驱动的优化迭代。

解析：自动化评测的价值不在“打一组分数”，而在于为不同版本提供同口径、可比较的证据，从而把“我觉得更好”变成“在同一组业务问题上确实更好”。

知识点：评测设计、版本比较、指标与人工结合

课程映射：`2_4_自动化评测答疑机器人的表现.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：自动化评测 -> 评测集设计 / 版本比较 / 评测闭环指导优化

## 四、补充单选题答案解析

### 19. 【中档】企业知识库已经完成向量化，但用户换一种口语化问法后，系统召回结果明显变差。以下哪项优化最有针对性？

标准答案：B

解析：题干的核心症状是“问法变化导致召回不稳”，这首先是检索表达问题，而不是生成参数问题。query rewrite 的价值就是把用户自然问法改写成更接近知识库表达的检索查询，从而提升召回鲁棒性。

错项分析：A 只影响生成随机性；C 是生成长度控制，不解决召回；D 会让切片过大、噪声更重，通常不是正向优化。

知识点：query rewrite 与召回鲁棒性

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 查询改写 / 召回优化

### 20. 【中档】某制度库当前切片粒度很大，且几乎没有重叠。系统虽然偶尔能命中相关段落，但回答经常遗漏限制条件。更合理的优化方向是？

标准答案：C

解析：题干说明已经能“偶尔命中相关段落”，但上下文条件经常缺失，这通常意味着 chunk 过大或边界切得不合理。缩小 chunk 并设置适度 overlap，能更好保留局部语义与前后条件。

错项分析：A 会进一步放大切片粗糙问题；B 直接塞整篇文档会带来更大噪声和上下文浪费；D 只调 `top_k`，没有触及切片质量这一根因。

知识点：chunk size / overlap 取舍

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 切片策略 / 召回质量

### 21. 【中档】RAG 系统已经能召回足够多的候选片段，但排在前面的结果经常不是最相关的，模型也因此引用了次优证据。此时最应该优先补哪一步？

标准答案：A

解析：当问题已经从“召回不到”变成“排序不准”时，更针对的手段是重排序。它的职责不是扩大候选，而是在候选中把最相关证据排到前面，让模型优先看到高质量上下文。

错项分析：B 容易掩盖根因；C 让生成更发散；D 删除 metadata 可能反而损失排序信号。

知识点：召回与重排序的职责边界

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 重排序 / 上下文构建优化

### 22. 【中档】财务报销制度中大量关键信息存在表格里，比如“城市等级-住宿上限-交通补贴”对应关系。为了提升问答准确率，以下哪种处理更合理？

标准答案：D

解析：表格类信息的关键在于行列关系和单元格对应关系，如果只保留截图或笼统摘要，往往会丢失可检索的结构。对表格做结构化展开或增强，才能让检索和生成阶段真正利用这些字段关系。

错项分析：A 不利于文本检索；B 直接删除关键信息；C 让模型猜表格规律，会增加幻觉风险。

知识点：表格增强的适用场景

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 表格内容增强 / 非纯文本知识处理

### 23. 【中档】你在比较两版问答系统，发现两版都能返回答案，但版本 B 更容易“答得像样，却没有证据支撑”。为了更快发现这一问题，最应该新增哪类评测维度？

标准答案：B

解析：题干要抓的不是“有没有答案”，而是“答案是否真由证据支持”。Faithfulness 一类指标正是用于衡量回答与检索证据之间的一致性，比只看耗时、文档量或 token 成本更贴近问题本身。

错项分析：A 属于前端体验；C 是知识库规模信息，不是回答质量指标；D 是成本指标，不能直接揭示证据一致性。

知识点：自动化评测中的证据一致性指标

课程映射：`2_4_自动化评测答疑机器人的表现.ipynb`

考纲映射：自动化评测 -> 评测维度设计 / 回答支撑性判断

### 24. 【拔高】某用户问题在知识库中没有明确证据，但业务又要求系统不能乱答，也不能直接沉默。最合理的处理方式是？

标准答案：C

解析：高质量 RAG 不等于“永远有答案”，而是“有证据才作答，没证据就清楚说明边界”。在证据不足时明确提示信息不够，并引导补充信息或人工确认，既控制风险，也保留服务连续性。

错项分析：A 和 B 都会放大无证据作答；D 盲目增大 `top_k` 只会把更多噪声拉进来，不代表能找到正确证据。

知识点：拒答与服务兜底策略

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 基于证据回答 / 拒答策略

### 25. 【拔高】团队新接入了一批制度文档后，用户反馈“很多问题突然答不上来了”。如果时间有限，只能优先排查一处，最合理的是？

标准答案：A

解析：文档新增后效果骤降，最常见的首因不是模型突然变差，而是文档解析、切片、入库或索引链路出了问题。先确认新文档是否真正进入可检索状态，比先改提示词或换模型更符合排障顺序。

错项分析：B 可能有帮助，但不该先于数据链路核对；C 成本高且定位不准；D 与题干问题无关。

知识点：检索失败时的优先排查顺序

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

### 26. 【拔高】团队同时改了切片策略、重排序模型和拒答阈值。上线后系统对一批原本能答的问题开始频繁拒答。若想最快判断问题主要出在“召回不足”还是“拒答阈值过严”，最合理的做法是？

标准答案：B

解析：这道题的关键不只是“做对照实验”，而是要把链路拆开看。只有在同一批问题上同时检查召回结果、重排序结果和最终拒答判定，才能判断到底是上游没有找到足够证据，还是下游阈值把本可回答的问题拦掉了。

错项分析：A 只看最终拒答率，无法定位根因；C 会继续引入新变量；D 直接关闭拒答虽然可能短期恢复表面通过率，但会掩盖真正问题并放大风险。

知识点：分层诊断与拒答归因

## 五、补充多选题答案解析

### 27. 【中档】关于 query rewrite 与召回鲁棒性，以下哪些说法更合理？

标准答案：ACD

解析：query rewrite 可以把用户口语化表达映射到更适合检索的查询形式，在多轮场景下还能结合上一轮语义补全问题。同时，它是否真的有效，仍需要通过统一评测集验证，而不是靠直觉判断。

错项分析：B 错在把 query rewrite 当成知识库替代品。改写只是优化查询表达，不能替代检索本身。

知识点：query rewrite、多轮追问与评测验证

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 查询改写 / 多轮检索优化

### 28. 【中档】关于切片、重排序和表格增强，以下哪些判断更合理？

标准答案：ABC

解析：重排序适用于“候选足够但顺序不准”的场景；表格如果只保留笼统摘要，常会丢失字段对应关系；chunk 和 overlap 则要结合文档结构与问题粒度来调。三者都体现了“不同问题对应不同优化点”。

错项分析：D 过于绝对。overlap 过大时会显著增加冗余和噪声，不是越大越好。

知识点：切片、重排序与表格增强的适用边界

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 切片优化 / 重排序 / 表格增强

### 29. 【中档】关于 RAG 自动化评测闭环，以下哪些做法更合理？

标准答案：ABD

解析：高质量评测闭环需要同批样本对比、覆盖关键质量维度，并保留人工抽样复核。自动化评测的价值在于支持持续迭代，而不是只在项目结束时做一次汇总。

错项分析：C 错在把评测当成一次性动作，这会让后续优化失去稳定判断基线。

知识点：自动化评测闭环

课程映射：`2_4_自动化评测答疑机器人的表现.ipynb`

考纲映射：自动化评测 -> 评测集复用 / 持续迭代

### 30. 【拔高】为了降低“无证据也继续作答”的风险，以下哪些组合策略更合理？

标准答案：BCD

解析：真正能降低无证据作答风险的，不是让模型“更自信”，而是同时约束生成、提升上下文质量并设计兜底路径。证据约束、切片过滤或重排序，以及证据不足时拒答或转人工，是更完整的组合方案。

错项分析：A 会放大主观生成，不会提升证据一致性，反而可能让错误答案更像真的。

知识点：证据约束、上下文清洗与兜底策略

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 基于证据回答 / 拒答 / 上下文优化

## 考点覆盖与复习建议

- 基础题优先复习参数作用、消息历史、系统提示词和 RAG 基础链路。

- 补充题要重点串起 query rewrite、chunk / overlap、表格增强、faithfulness 评测和证据兜底之间的联动关系。
