## 一、单选题答案解析

### 1. 在调用大模型生成答复时，以下哪个参数最直接影响输出的随机性？

标准答案：B

解析：`temperature` 用于调节采样随机性，值越高通常输出越发散。`model` 决定模型版本，`messages` 承载上下文，`stream` 决定是否流式返回。

知识点：大模型调用参数中的随机性控制

课程映射：`2_1_用大模型构建新人答疑机器人.ipynb`

考纲映射：大模型应用开发 -> 基本 API 参数如 `model`、`temperature`、`top_p`

### 2. 在构建支持连续追问的答疑机器人时，最关键的做法是：

标准答案：C

解析：多轮对话依赖历史消息来维持上下文一致性，否则模型无法理解代词、省略和追问语义。其余选项都不能直接保证上下文连续性。

知识点：多轮对话中的上下文与历史消息管理

课程映射：`2_1_用大模型构建新人答疑机器人.ipynb`

考纲映射：大模型应用开发 -> 理解消息与对话历史

### 3. 在提示词工程中，系统角色提示词最核心的作用是：

标准答案：A

解析：系统角色提示词用于定义身份、边界和输出风格，是较稳定的高优先级约束。它既不能替代用户输入，也不只和长度或微调相关。

知识点：系统角色提示词的作用与优先级

课程映射：`2_3_优化提示词改善回答质量.ipynb`

考纲映射：大模型提示词工程 -> 理解系统角色提示词的作用

### 4. 以下哪项最符合课程中“利用大模型处理各类任务”的思路？

标准答案：C

解析：课程强调大模型不仅能问答，还能承担分类、审阅、修订等通用语言任务。A、B、D 都把模型能力限制得过窄。

知识点：大模型在通用语言任务中的应用范围

课程映射：`2_3_优化提示词改善回答质量.ipynb`

### 5. 一个基础 RAG 流程中，较合理的顺序是：

标准答案：B

解析：RAG 通常先解析原始文档，再切片建立检索单元，随后完成召回与重排序。顺序颠倒会导致检索对象无法稳定构建。

知识点：RAG 基础链路的标准流程

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`

### 6. 关于 RAGAS 的描述，哪一项最准确？

标准答案：B

解析：RAGAS 是面向 RAG 应用评测的一套指标体系，用来帮助量化回答质量、检索支撑等方面表现。它既不是部署工具，也不负责知识库构建。

## 三、场景题参考答案

### 14. 场景题：新人答疑机器人“能答但答不准”

参考答案：
1. 先补齐基础 RAG 流程，确保知识库解析、切片、召回、重排序链路完整，这是答不上来的第一层原因。
2. 再优化 Prompt 设计，包括系统角色、任务描述和输出模板，提升不同问法下的理解稳定性。
3. 对召回链路做进一步优化，如句子窗口检索、标题改写和表格增强，解决“只命中零碎片段”的问题。
4. 最后引入自动化评测，用统一样本和指标比较优化前后的效果，避免只凭主观感受判断。

解析：该场景同时涉及 `2.2`、`2.3`、`2.4`、`2.5`。先保证基础链路可用，再做提示词与检索优化，最后用评测闭环验证，是更稳妥的工程路径。

知识点：RAG 问答系统的诊断、优化与评测闭环

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_3_优化提示词改善回答质量.ipynb`、`2_4_自动化评测答疑机器人的表现.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> RAG 核心要素、RAG 优化、RAG 自动化评测；大模型提示词工程 -> 构建有效提示词

### 15. 场景题：如何用自动化评测指导迭代

参考答案：
1. 评什么：围绕代表性问答样本评估回答质量与检索支撑质量，重点关注 RAGAS 类指标以及问题覆盖面。
2. 怎么比：在同一批测试集上分别运行版本 A 和版本 B，比较回答正确性、检索相关性和整体稳定性，避免样本不一致造成偏差。
3. 如何决策：若版本 B 在主要指标上稳定优于版本 A，则保留优化方案；若部分问题仍表现较差，再回到切片策略、召回策略或提示词设计继续细化。

解析：自动化评测的关键不只是“打分”，而是让优化动作能被同口径比较，并形成持续迭代闭环。这也是 C2 从构建到优化再到评测的核心方法论。

知识点：自动化评测驱动优化迭代

课程映射：`2_3_优化提示词改善回答质量.ipynb`、`2_4_自动化评测答疑机器人的表现.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型提示词工程 -> 系统角色提示词；大模型检索增强 -> RAGAS 指标体系、RAG 系统评测方法、RAG 优化方法

## 考点覆盖与复习建议

- 重点覆盖大模型应用开发、提示词工程、RAG 基础链路、RAG 优化与自动化评测。
- 单选题优先复习参数控制、系统提示词和基础 RAG 流程；多选题重点回看提示词设计方法与 RAG 评测框架。
- 场景题复盘时，建议按“链路诊断 -> 提示词优化 -> 检索优化 -> 评测验证”的顺序整理答案思路。
