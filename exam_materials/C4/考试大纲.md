# C4 考试大纲

本文档用于支撑 `C4` 模块题库升级，不仅整理《阿里云大模型 ACP 考试大纲》中与“交付上线”相关的核心要求，也明确每类考点适合出的题型和难度层级，作为 `模拟题题目.md` 与 `模拟题答案解析.md` 的命题基线。

## 一、模块定位与权重

- 模块名称：`C4 交付上线`
- 对应教程：`4_0` 到 `4_4`
- 对应考试大纲板块：模型定制与优化、生产环境应用实践、安全与合规
- 模块特征：更强调工程判断、方案选型、上线治理和纵深防御

按现有教程映射，`C4` 主要覆盖以下能力群组：

| 模块 | 对应教程 | 备考重点 |
| --- | --- | --- |
| 蒸馏与微调边界 | `4_1` | 任务适配性、数据质量、评测优先级、合规边界 |
| 模型部署与发布选型 | `4_0`、`4_2` | API、百炼、vLLM、PAI-EAS、FC、ACK、ECS 的选型逻辑 |
| 生产环境应用实践 | `4_3` | 需求分析、SLO、TTFT、缓存、降级、稳定性与成本 |
| 安全合规与纵深防御 | `4_4` | 提示词注入、工具风险、黑名单、AI 安全护栏、权限模型 |

## 三、命题难度分布建议

建议将 `C4` 题库组织为一套 `32` 题的标准强化卷：

- 单选题 `18`：基础 `7`、中档 `8`、拔高 `3`
- 多选题 `14`：基础 `5`、中档 `6`、拔高 `3`

对应命题重点如下：

| 难度 | 目标 | 典型题型 |
| --- | --- | --- |
| 基础 | 覆盖蒸馏、部署、SLO、安全的主干边界 | TTFT、功能性需求、注入攻击识别 |
| 中档 | 引入条件、约束和工程取舍，考选型与优先级 | API 与自部署、FC 与 EAS、评测指标、优化手段 |
| 拔高 | 强调跨模块的完整链路设计与纵深防御 | 蒸馏与在线生成组合、工具权限、上线治理综合题 |

## 四、易错方向

- 把“适合蒸馏”和“适合 RAG / 在线大模型”混为一谈。
- 把“能起服务”和“能承接生产流量”混为一谈。
- 只记性能指标名称，不会判断需求优先级和生产治理顺序。
- 把 AI 安全护栏误当成权限模型和工具鉴权的替代品。
- 只会记云服务名称，不会根据流量、预算、冷启动和运维能力做部署选型。

## 五、从大纲到题目的落地原则

- 大纲先定义“考什么、考多深、适合出什么难度”。
- 题目再落实为“在什么场景下考、如何形成区分度、如何避免送分项”。
- 解析最后落实为“为什么对、为什么错、应该回到哪个知识点复习”。

这意味着 `C4` 不只是交付上线知识清单，也应当成为高权重工程模块的命题样板。
