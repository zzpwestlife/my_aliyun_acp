## 答案与解析

### 单选题

**1. B**　百炼提供 OpenAI 兼容模式，`base_url` 固定为 `https://dashscope.aliyuncs.com/compatible-mode/v1`，配合 `DASHSCOPE_API_KEY` 使用。

**2. B**　密钥绝不能明文入代码或上传公开仓库，环境变量/配置文件加载是基本安全实践。

**3. B**　`stream=True` 开启流式输出，逐 Token 返回。

**4. B**　创意类文本需要多样性，应适当调高 temperature；事实问答、抽取类任务则调低（接近 0）。

**5. B**　top_p（核采样）只在累计概率达到阈值的最小 Token 集合中采样。`top_p=0.1` 表示仅在概率最高、累计到 10% 的极少数 Token 中选择，输出非常确定。

**6. C**　`===`、`---` 多用于**分隔段落**；`【】`、`<< >>`、`###`、XML 标签多用于**标识关键要素**。这是官方课程明确区分的两类用途。

**7. C**　ToT、GoT 都是在思维链（CoT）"让模型分步思考"基础上的结构化扩展。

**8. B**　提供示例样本即 Few-shot；不给示例为 Zero-shot。

**9. B**　Meta Prompting 是用大模型来迭代优化提示词，包括定性指导和基于“参考答案”的自动化迭代，是 2025 年 7 月（V2.1.0）引入的考点。

**10. B**　上下文工程把 RAG、Prompt、Tool、Memory 整合为系统化方法论，本质是“为模型精准构建和动态管理上下文”。

**11. B**　标准写法是 `SimpleDirectoryReader('./docs').load_data()`。

**12. B**　持久化用 `index.storage_context.persist(路径)`。

**13. B**　Retrieval = 检索；Generation 才负责生成答案。

**14. B**　K 过小会漏掉本应召回的相关切片，使答案不完整；但 K 过大又可能超长被截断，需折中（官方推荐"智能拼装"）。

**15. B**　开启拒识可约束模型仅基于知识库回答，抑制其调用自身通用知识。

**16. B**　RAGAS 是 RAG 专用的自动化评测指标体系；BLEU/ROUGE 偏翻译摘要，Perplexity 偏语言模型困惑度。

**17. B**　微调适合"有足量标注数据 + 需稳定特定输出"的专有任务；实时信息类应走 RAG/工具调用。

**18. B**　SFT 数据通常是带 system/user/assistant 角色的消息对（messages 结构）。

**19. B**　2026 版（V2.4.2）将重点从“调参数”转向“迁移能力”，通过蒸馏让小模型掌握大模型的专业能力，更贴合工程现实。

**20. A**　Qwen Code 是通义千问团队开源的终端 AI 智能体，2026 版（V2.4.0）新增其实践章节。

**21. D**　训练耗时是资源/效率指标，不直接衡量分类效果；Accuracy、Recall、F1 才是分类效果指标。

**22. B**　Agent 的核心循环是“思考-行动-观察”。

**23. B**　长期记忆对应系统持久化信息（业务历史、知识库），通常存于外部向量数据库/文档库。

**24. B**　Skill 把验证过的能力固化为可复用流程；Memory 负责记忆，二者不同。

**25. B**　工具调用让模型能访问外部接口/工具，获取实时信息或执行动作，突破模型自身知识边界。

**26. A**　HITL（Human-in-the-Loop）将人类决策嵌入关键节点，构建可信 AI 系统。

**27. A**　vLLM 是主流高吞吐大模型推理框架，官方课程明确要求掌握其部署。

**28. A**　函数计算 FC 事件驱动、按量付费、免运维，适合快速发布轻量 AI 助手。

**29. A**　2026 版（V2.0.8）引入阿里云“AI 安全护栏”产品做多层防护。

**30. A**　可观测性新增内容覆盖 Trace、日志、指标监控，帮助应用“跑得稳”。

**31. B**　一律用最大模型且关缓存会显著推高成本与延迟，不合理；其余均为合理的降本增效手段。

### 多选题

**32. ABD**　temperature、top_p 直接控制随机性；固定 seed 可影响可复现性（同 seed 结果一致）。max_tokens 只控制长度，不影响随机性。

**33. ABC**　超长上下文会触及长度限制、拉低效率与抬高成本，并稀释关键信息；D 与事实无关。

**34. ABC**　任务目标、上下文、输出格式（及角色、样例）是核心要素；模型参数量不是提示词要素。

**35. ABD**　句子窗口检索、自动合并检索、相似度阈值均为有效优化；C 错——K 并非越大越好，过大易超长被截断。

**36. ABC**　边界清晰、角色明确、风格与标签一致都是高质量数据要求；D 错——低质乱码会把模型训坏。

**37. ABC**　蒸馏让小模型获得接近大模型的专业能力、更省成本易落地，并可借 Qwen Code 实现；D 过于绝对，小模型不可能在所有任务上全面超越大模型。

**38. ABC**　工具调用用于查实时数据、精确计算、检索内部库；D 错——工具调用不会修改模型权重。

**39. ABD**　分层规划、共创协作是常见模式，AgentScope 可实现评测闭环；C 是故障而非协作模式。

**40. ABCD**　百炼、FC、PAI-EAS、ECS/容器均为官方列出的部署方案。

**41. ABC**　安全护栏检测、黑名单合规检测、注入攻击防护都正确；D 错——不过滤直接拼接是高危做法。

**42. ABCD**　上下文工程正是统筹 RAG、Prompt、Tool、Memory 四要素。

**43. ABC**　超时重试、降级 fallback、可观测性都能提升稳定性；D 错——关闭监控会让问题无法被及时发现。
