# 模拟题排版优化设计

## 背景

现有模拟题内容已经可用，但在阅读体验上仍偏“原始 Markdown 文档”，试卷感、复盘感和场景题书写空间都不够明显。
本次优化只调整排版结构与阅读体验，不修改题目知识点、答案结论和题量。

## 目标

- 让三份题库看起来像一套统一资料，而不是三个松散文档
- 提升做题时的层级感、留白和分区感
- 提升复盘时的答案可扫读性
- 给场景题增加更明确的作答提示和留白结构

## 影响范围

本次只修改以下三个文件：

- `.trae/specs/generate-c2-mock-exam/mock_exam_questions_only.md`
- `.trae/specs/generate-c2-mock-exam/mock_exam_answers_only.md`
- `.trae/specs/generate-c2-mock-exam/mock_exam_timed_full.md`

不修改：

- `.trae/specs/generate-c2-mock-exam/mock_exam.md`
- 题目答案本身
- 题目数量、知识覆盖范围、考纲映射逻辑

## 设计方案

### 1. 统一视觉结构

三份文件统一采用相同的排版骨架：

- 顶部保留“用途说明”
- 大章节使用一致的标题样式
- 每道题之间加入更清晰的分隔
- 题号、题干、选项、答案信息保持固定顺序

这样用户在三个版本之间切换时，不需要重新适应阅读结构。

### 2. 试卷感优化

重点优化 `mock_exam_questions_only.md` 和 `mock_exam_timed_full.md`：

- 增强“说明区 / 答题区 / 正文区”的边界
- 在每个题型区块前增加小节提示
- 每道题与下一题之间留出更明显空白或分隔线
- 计时模拟版补强“答题卡”和“做完再看答案”的视觉提醒

目标是让它更接近“可直接练习的试卷”。

### 3. 答案区优化

重点优化 `mock_exam_answers_only.md`：

- 每题采用固定的答案信息顺序：
  - 题型
  - 标准答案 / 参考答案
  - 解析
  - 课程映射
  - 考纲映射
- 通过标签化写法增强扫读体验
- 让“答案”和“解析”在视觉上比“映射信息”更突出

目标是让这份文档更像“复盘手册”。

### 4. 场景题优化

重点优化 `mock_exam_questions_only.md` 和 `mock_exam_timed_full.md` 的场景题区：

- 增加“作答提示”
- 增加建议作答结构，例如：
  - 问题判断
  - 优先优化项
  - 验证方法
- 增加留白区，方便直接在文档里思考或誊写答案

在答案版中：

- 将场景题参考答案保留为分点结构
- 让解析与参考答案区分更明显

### 5. 风格边界

本次优化遵循以下边界：

- 使用纯 Markdown 能稳定表达的结构
- 不引入 HTML/CSS 自定义样式
- 不引入 emoji、复杂装饰符号或花哨版式
- 不改变题目原义

## 预期结果

优化后：

- 只出题版更适合闭卷刷题
- 答案单独版更适合错题复盘
- 计时模拟版更像正式练习卷
- 三份文件的整体风格保持统一

## 验收标准

- 所有题目内容保持不变
- 选项继续保持单独成行
- 场景题具备明确的作答提示
- 答案版信息层次明显优于当前版本
- 三份文件通过 Markdown 诊断
