# 大模型 ACP 考前速记与易错点

## 最该先背的判断规则

1. RAG 解决“知识库、长文档、事实更新、输入超长、内部资料问答”；微调解决“稳定任务风格/领域能力”，不是频繁更新知识的首选。
2. 检索效果差先看文档解析、切片、Embedding、召回数量、相似度阈值、rerank、query 改写，不要先调 temperature。
3. 生成格式不稳看提示词输出格式、样例、temperature、seed、max_token；内容缺知识看 RAG。
4. system 角色用于设定身份、规则和安全边界；user 才是用户输入；assistant 是模型回复。
5. temperature 越高越发散，top_p 越小越保守，seed 固定提升一致性，stream 改善等待体验但不提升模型能力。
6. RAGAS 中 context_recall/context_precision 看检索上下文，faithfulness 看回答是否忠于上下文，answer_correctness 看最终答案正确性。
7. 微调训练循环常见顺序：model.train -> 前向传播 -> loss -> zero_grad -> backward -> optimizer.step。
8. 早停主要看验证集，不看训练集自嗨；训练损失降、验证损失升是过拟合。
9. 内容安全要尽早拦截输入，输出前也要审核；API Key 放环境变量；备案/隐私授权不能跳过。
10. Agent 工具调用错，优先检查工具描述、参数 schema、Planner/路由边界，而不是盲目换大模型。

## 按模块背诵

- RAG 核心流程、检索优化、文档解析/切片：RAG 题优先判断问题发生在“解析、切片、向量化、召回、重排、生成”哪一层，不要把检索问题误当成单纯换大模型或调 temperature。
- RAG 自动化评测与 RAGAS 指标：RAGAS 题要分清检索指标和生成指标：context_recall/context_precision 看上下文，faithfulness/answer_correctness 更偏生成答案质量。
- 微调概念、数据构建、训练参数与评测：微调题先抓训练闭环：数据质量、前向传播、损失、反向传播、梯度更新、验证集评估；不要把推理参数和训练参数混用。
- 提示词框架、系统角色与任务设计：提示词题看是否明确了角色、任务、上下文、约束、输出格式和样例；含糊、只喊口号、没有格式约束的选项通常不优。
- API 调用参数、消息角色与对话历史：API 题要区分 model、messages、system/user/assistant、temperature/top_p、stream、seed、max_token 的职责。
- 大模型适用场景与应用开发基础：应用开发题重点是判断大模型适合语义理解和生成，精确计算、确定性统计、传统数据结构题通常不是最佳答案。
- Assistant API、智能体机制与工具调用：Agent 题要抓“任务拆解、工具描述、路由决策、函数调用、结果提交”，工具调用错常见原因是描述或参数 schema 不清。
- 多模态内容生产与行业应用：多模态题要分清文本、图像、语音、视频各自模型/工具的边界，避免把 TTS、视频剪辑、视觉理解混为一谈。
- 内容安全、合规检查与应用安全：安全合规题优先考虑输入过滤、权限控制、输出审核、备案、隐私授权和 API Key 安全；“先上线再说”基本必错。
- 部署发布、性能与生产实践：生产实践题看稳定性、降级、成本、延迟和资源约束；流式输出改善体验，不等于降低模型计算量。

## 本题库已删除的干扰题类型

- 缺图或抽取损坏题。
- 纯 Python 语法/数据结构题。
- 纯 ffmpeg/moviepy 工具 trivia。
- GPU Spot/包年包月这类纯云资源计费题。
- 明显过时或模型名称错误题。
