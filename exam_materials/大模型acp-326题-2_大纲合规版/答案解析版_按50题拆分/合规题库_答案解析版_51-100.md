#### 第 51 题 [单选题]

在 RAG 系统中，将文档拆分成多个分块（chunks）的主要目的是什么？

A. 适配大模型的输入限制，并提高检索的准确性和相关性。

B. 减小存储空间，提高检索效率。

C. 方便用户浏览，提升用户体验。

D. 简化文档预处理流程，降低开发成本。

**答案：`A`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

**相关知识点：** RAG 的关键链路是：文档解析 -> 切片/补上下文 -> embedding 向量化 -> 向量库召回 -> rerank/过滤 -> 拼接上下文生成。答题时先定位问题发生在哪一层。 RAG = 外部知识 + 检索相关片段 + 拼接上下文 + 生成答案。


**原解析补充：**

这是分块的两个核心目的：1.适配模型有限的上下文窗口,2.使检索结果更精准（小分块能更精确匹配查询）

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 正确：适配大模型的输入限制，并提高检索的准确性和相关性。
  - 原因：正确，因为该选项直接改善 RAG 的解析、切片、向量化、召回、重排或上下文质量。
- B 不选：减小存储空间，提高检索效率。
  - 原因：分块确实有一定检索效率收益，但"减小存储空间"并非核心目的；分块的本质是适配上下文窗口限制并提升检索精准度，而非节省存储。
- C 不选：方便用户浏览，提升用户体验。
  - 原因：分块是为了让模型能召回并利用知识，"方便用户浏览"是界面展示层的目标，与 RAG 文档切片的技术动因无关。
- D 不选：简化文档预处理流程，降低开发成本。
  - 原因：分块恰恰增加了预处理复杂度（需要选择切片策略、设定 overlap），不会简化流程；该选项方向相反。

**易错点分析：**

易错点：不要把所有问题都归因于“大模型不够强”。如果问题出在文档解析、切片、召回或重排，调 temperature 或换生成模型通常不是首选。

#### 第 52 题 [单选题]

数据预处理中，将分类任务的标签转化为模型可理解的数字形式称为什么？

A. 文本标准化

B. 词嵌入（Word Embedding）

C. 特征缩放

D. 标签编码（Label Encoding）

**答案：`D`**

**考点：** 大模型微调与数据处理 / 分类标签预处理、标签编码

**详细解析：**

**相关知识点：** 分类任务中的原始标签通常是字符串或业务类别，例如“通过/拒绝”“正向/负向”。训练或评估时，模型和损失函数需要可计算的数值标签，因此要把类别标签映射成数字 ID，这个过程就是标签编码。它解决的是“目标标签如何表示”的问题，不是文本内容如何向量化，也不是数值特征如何缩放。

**原解析补充：**

将类别标签转换为数字，是分类任务的标准预处理步骤。

**解题逻辑：** 题干关键词是“分类任务的标签”和“转化为模型可理解的数字形式”。只要看到“类别 -> 数字编号”，就优先对应 Label Encoding；不要被 Word Embedding 里的“向量/数字表示”带偏，因为词嵌入处理的是文本 token 或词语语义，不是类别标签。

**逐选项原因：**
- A 不选：文本标准化
  - 原因：文本标准化通常是大小写、全半角、符号、空格、繁简或噪声清洗，处理的是输入文本格式，不负责把分类标签映射成数字。
- B 不选：词嵌入（Word Embedding）
  - 原因：词嵌入是把词、句子或文本片段转成稠密向量，用于语义表示；题目问的是分类标签的编码，不是文本语义向量化。
- C 不选：特征缩放
  - 原因：特征缩放用于把连续数值特征调整到相近尺度，例如归一化或标准化；它不用于类别标签到数字 ID 的映射。
- D 正确：标签编码（Label Encoding）
  - 原因：标签编码就是把类别标签转换为模型可处理的数字形式，正好命中题干“分类任务标签转数字”的定义。

**易错点分析：**

易错点：看到“模型可理解的数字形式”容易误选词嵌入。区分方法是看对象：输入文本变向量是 embedding；类别标签变编号是 label encoding；连续数值特征调尺度是 feature scaling。

**原题号：** 58

#### 第 53 题 [单选题]

A. 记录不安全信息，用于后续模型改进和安全策略优化。

B. 直接将原始输出返回给用户。

C. 过滤掉不安全信息，返回剩余安全内容。

D. 向用户返回提示信息，告知其输出内容存在风险。

**答案：`B`**

**考点：** 生产环境应用实践 / 内容安全、合规检查与应用安全

**详细解析：**

**相关知识点：** 生产安全题优先考虑 API Key 安全、输入过滤、输出审核、权限控制、隐私授权、备案、日志追踪和风险兜底。 Production = 安全、合规、部署、性能、成本、监控、降级。


**原解析补充：**

直接返回不安全信息可能造成伦理/法律风险如传播有害内容

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 不选：记录不安全信息，用于后续模型改进和安全策略优化。
  - 原因：记录不安全案例是合规且正确的做法，可用于持续优化安全策略，不存在直接向用户输出有害内容的风险。
- B 正确：直接将原始输出返回给用户。
  - 原因：发现输出包含不安全信息后直接返回给用户，会将有害内容传播给用户，既违反合规要求也存在法律风险，是最不推荐的做法。
- C 不选：过滤掉不安全信息，返回剩余安全内容。
  - 原因：对不安全内容进行过滤后再返回，是合理的内容安全处理方案，属于推荐做法而非错误做法。
- D 不选：向用户返回提示信息，告知其输出内容存在风险。
  - 原因：告知用户存在风险内容并拦截输出，是透明且负责任的安全处理方式，属于推荐做法。

**易错点分析：**

易错点：“先上线后备案”“直接返回风险内容”“把 Key 写进代码”通常都是错误方向。

**相关知识点：** 微调是用任务数据继续训练预训练模型，核心关注数据质量、训练循环、损失函数、梯度、学习率、batch size、epoch、验证集和过拟合/欠拟合。 Fine-tuning = 用任务数据更新模型参数，适合稳定任务适配。


**原解析补充：**

微调大语言模型的数据预处理流程中，旨在减少文本中的噪声、提高数据质量的步骤是文本清洗

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

- A 不选：序列填充
  - 原因：序列填充（padding）是为了将不同长度的序列对齐到统一长度以便批量训练，目的是格式对齐，而非去除文本噪声。

- C 不选：特征选择
  - 原因：特征选择是从已有特征中筛选对任务有效的子集，用于传统 ML 场景；在大模型微调的文本预处理中不是标准步骤，也不直接解决噪声问题。
- D 不选：数据增强
  - 原因：数据增强是通过同义替换、回译、插入等方式扩充数据量，目的是增加训练样本多样性，而非减少原有数据中的噪声。

**易错点分析：**

易错点：微调不是知识库更新的低成本替代；频繁变化的事实知识通常优先用 RAG。训练题要分清 loss、backward、step 的职责。
