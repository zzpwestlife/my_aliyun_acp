#### 第 101 题 [多选题]

当你在使用一款基于大模型构建的创意助手，要求它一次性产生 10 条宣传文案时，发现生成的文案千篇一律，缺乏多样性。为了在保证一定多样性的同时避免模型生成不符合主题的内容，可以优先考虑以下哪些改进措施？

A. 使用模糊的指令，让模型自由发挥。

B. 检查提示词是否过于局限，尝试在提示词中明确指定多个角度方向（如情感、风格、视角等）来扩展其灵活性。

C. 固定 top_p=0.1 以确保最大范围的词汇选择。

D. 小幅调高 temperature 以保证生成内容的多样性。

**答案：`BD`**

**考点：** 大模型应用开发 / API 调用参数、消息角色与对话历史

**详细解析：**

**相关知识点：** API 调用题要分清 model、messages、system/user/assistant、temperature、top_p、stream、seed、max_tokens 等参数各自作用。 API = 参数、消息角色、流式/批量、对话历史和适用边界。


**原解析补充：**

B 提供结构化多样性引导，D 增加适当随机性，两者结合可优化生成效果

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 不选：使用模糊的指令，让模型自由发挥。
  - 原因：不选，因为"模糊指令"反而会让模型在没有方向约束的情况下重复已有模式，无法引导输出多角度内容，与解决"千篇一律"问题背道而驰。
- B 正确：检查提示词是否过于局限，尝试在提示词中明确指定多个角度方向（如情感、风格、视角等）来扩展其灵活性。
  - 原因：正确，因为提示词中明确给出多个角度（如情感、风格、视角），直接指示模型从不同维度生成文案，是解决"千篇一律"最直接的结构化手段。
- C 不选：固定 top_p=0.1 以确保最大范围的词汇选择。
  - 原因：不选，因为 top_p=0.1 是极低的核采样，意味着只从概率最高的少数词中选择，实际上大幅压缩词汇多样性，描述与效果相反，无法扩大词汇选择范围。
- D 正确：小幅调高 temperature 以保证生成内容的多样性。
  - 原因：正确，因为适当提高 temperature 会让模型在 token 采样时分布更均匀，减少重复相似输出，是增加文案多样性的标准参数手段。

**易错点分析：**

易错点：temperature/top_p 控制随机性，max_tokens 控制长度，stream 控制返回方式，seed 控制一致性，system 不是用户提问。

**原题号：** 107

#### 第 102 题 [多选题]

用户多次反馈 “AI 回答中有捏造内容”，排查后发现是当知识库没有相关内容时，大模型直接虚构答案。为减少此情况，你可以怎么做？

A. 在百炼 RAG 应用中将回答范围设置为 “仅知识库范围”

B. 调高 temperature 值

C. 使用 ReRank 对检索结果再次排序

D. 使用提示词，无匹配就明确回复 “无法回答”

**答案：`AD`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

**相关知识点：** RAG 的关键链路是：文档解析 -> 切片/补上下文 -> embedding 向量化 -> 向量库召回 -> rerank/过滤 -> 拼接上下文生成。答题时先定位问题发生在哪一层。 RAG = 外部知识 + 检索相关片段 + 拼接上下文 + 生成答案。


**原解析补充：**

A 适用于严格依赖知识库的场景，D 作为兜底策略，两者均可减少幻觉。C 可辅助提升检索质量，但非必需；B 完全错误

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 正确：在百炼 RAG 应用中将回答范围设置为 “仅知识库范围”
  - 原因：正确，因为将回答范围限定为”仅知识库范围”后，知识库无匹配时模型不会自行补充，从根源上阻止了虚构答案的产生。
- B 不选：调高 temperature 值
  - 原因：不选，因为调高 temperature 只是提升输出的随机性，对"知识库无内容时模型虚构答案"这一根因没有任何约束作用，反而可能生成更多偏离事实的内容。
- C 不选：使用 ReRank 对检索结果再次排序
  - 原因：不选，因为 ReRank 只对已召回的内容重新排序，无法解决"知识库本就没有相关内容时模型直接虚构"的问题——根因在于生成阶段缺乏约束，而非检索排序。
- D 正确：使用提示词，无匹配就明确回复 “无法回答”
  - 原因：正确，因为在提示词中明确指令”无匹配则回复无法回答”，给模型设定了兜底行为约束，有效防止知识库无内容时的虚构输出。

**易错点分析：**

易错点：不要把所有问题都归因于“大模型不够强”。如果问题出在文档解析、切片、召回或重排，调 temperature 或换生成模型通常不是首选。

**原题号：** 108

#### 第 103 题 [多选题]

某电商公司需开发一个客服机器人处理 “退换货政策咨询” 和 “商品推荐” 任务，下列哪两种角色应该深度参与提示词设计？

A. 产品经理（定义用户场景和业务流程）

B. 前端开发工程师（负责对话界面实现）

C. 算法工程师（擅长模型调优和参数调整）

D. 客服部门主管（熟悉用户高频问题和政策细节）

**考点：** 大模型提示词工程 / 提示词框架、系统角色与任务设计

**详细解析：**

**相关知识点：** 提示词工程的核心是把角色、任务、上下文、约束、输出格式、样例写清楚；好的提示词通常具体、可执行、可解析。 Prompt = 把角色、任务、上下文、约束、输出格式和样例讲清楚。


**原解析补充：**

产品经理和客服主管是提示词设计的核心角色，分别从流程和内容层面保证机器人符合业务需求。其他角色（B C）更多是执行或技术支持，无需深度参与设计。

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 正确：产品经理（定义用户场景和业务流程）
  - 原因：正确，因为产品经理负责梳理用户场景与业务流程，能为提示词提供任务边界、对话目标和输出要求，是提示词框架设计的关键视角。
- B 不选：前端开发工程师（负责对话界面实现）
  - 原因：不选，因为前端工程师负责界面渲染与交互实现，与提示词的内容设计（角色、任务、约束）无直接关联，不是提示词设计的核心参与者。
- C 不选：算法工程师（擅长模型调优和参数调整）
  - 原因：不选，因为算法工程师擅长模型参数与微调，对退换货政策内容和用户对话场景缺乏业务理解，不适合主导提示词的角色与任务设计。
- D 正确：客服部门主管（熟悉用户高频问题和政策细节）
  - 原因：正确，因为客服主管掌握真实用户高频问题和退换货政策细节，能直接提供提示词所需的业务知识、约束规则和典型样例。

**易错点分析：**

易错点：只写“请做好/请准确”不算好提示词。考试更偏向选择具体、结构化、带格式和样例的表达。

#### 第 104 题 [多选题]

构建高质量的数学题微调数据集应包含哪些要素？

A. 包含详细分步推导

B. 统一使用 LaTeX 公式或 Markdown 格式输出公式

C. 大量的数学题解题数据

D. 覆盖多种解题思路

**答案：`ABD`**

**考点：** 大模型微调 / 微调概念、数据构建、训练参数与评测

**详细解析：**

**相关知识点：** 微调是用任务数据继续训练预训练模型，核心关注数据质量、训练循环、损失函数、梯度、学习率、batch size、epoch、验证集和过拟合/欠拟合。 Fine-tuning = 用任务数据更新模型参数，适合稳定任务适配。


**原解析补充：**

C 是常见误区，但“数据质量 > 数据量

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 正确：包含详细分步推导
  - 原因：正确，因为数学微调数据的核心质量要素是推理过程的完整性，分步推导让模型学到解题的中间思维步骤，而非只记住答案。
- B 正确：统一使用 LaTeX 公式或 Markdown 格式输出公式
  - 原因：正确，因为格式统一能让模型在训练时学到一致的公式表示规范，避免格式混乱导致的输出不稳定，提升数学表达的准确性和可解析性。
- C 不选：大量的数学题解题数据
  - 原因：不选，因为微调数据的关键是质量而非数量——"大量"本身不构成高质量数据集的要素，低质量数据堆砌反而会引入噪声，损害模型表现。
- D 正确：覆盖多种解题思路
  - 原因：正确，因为覆盖多种解题思路能提升数据集的多样性，让模型学到不同解法路径，增强泛化能力，避免只会一种套路的过拟合。

**易错点分析：**

易错点：微调不是知识库更新的低成本替代；频繁变化的事实知识通常优先用 RAG。训练题要分清 loss、backward、step 的职责。

#### 第 106 题 [多选题]

关于 temperature 和 top_p 参数的作用，以下哪三个描述是正确的?

**相关知识点：** API 调用题要分清 model、messages、system/user/assistant、temperature、top_p、stream、seed、max_tokens 等参数各自作用。 API = 参数、消息角色、流式/批量、对话历史和适用边界。


**原解析补充：**

B 错 低 temperature → 稳定性↑，多样性↓高 temperature → 稳定性↓，多样性↑

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 正确：较低的 top_p 可以提高生成内容的稳定性
  - 原因：正确，因为较低的 top_p 将候选词限定在概率最高的少数 token 中，减少了低概率词被采样的机会，输出更集中稳定。

  - 原因：不选，因为这一描述与事实相反——较低的 temperature 会使概率分布更尖锐，模型倾向于选择最高概率的 token，输出更单一稳定，多样性降低而非增加。

  - 原因：正确，因为较高的 temperature 使 token 概率分布趋于平坦，低概率词被采样的机会增加，输出更多样、更有创意。

#### 第 107 题 [多选题]

以下哪些方案，有助于提升大模型应用的用户体验？

A. 等全部内容生成后一次性返回

B. 展示任务处理进度

C. 生成的内容流式输出

D. 将任务分解为多个步骤，并且将分步执行过程展示给用户

**答案：`BCD`**

**考点：** 生产环境应用实践 / 部署发布、性能与生产实践

**详细解析：**

**相关知识点：** 生产实践题重点看延迟、成本、稳定性、降级、缓存、批处理、流式输出、资源限制和工程可维护性。 Production = 安全、合规、部署、性能、成本、监控、降级。


**原解析补充：**

提升用户体验的核心是 减少等待焦虑 和 增强交互透明性

**解题逻辑：** 先看题目问的是“最合适/不推荐/主要作用/哪些措施”，再定位到具体环节。单选题通常选最直接解决根因的一项；多选题则保留所有能直接改善该环节的措施，排除只改善无关指标、过度泛化或跳过必要流程的选项。

**逐选项原因：**
- A 不选：等全部内容生成后一次性返回
  - 原因：不选，因为"等全部生成后再返回"会造成用户长时间等待白屏，无任何进度反馈，显著降低体验，是应当避免的做法。
- B 正确：展示任务处理进度
  - 原因：正确，因为进度展示让用户感知到系统正在工作，减少等待焦虑，是提升感知体验的直接手段，尤其适合耗时较长的任务。
- C 正确：生成的内容流式输出
  - 原因：正确，因为流式输出（streaming）让用户看到内容逐步呈现，大幅缩短感知到的"首字节"等待时间，显著改善交互体验。
- D 正确：将任务分解为多个步骤，并且将分步执行过程展示给用户
  - 原因：正确，因为将复杂任务拆分并逐步展示执行过程，增强了交互透明性，用户能了解当前进展，避免"黑盒"等待感。

**易错点分析：**

易错点：流式输出改善用户等待体验，但不等于降低模型计算成本；缓存/批处理/降级才常用于成本和稳定性优化。

#### 第 108 题 [多选题]

关于 Loss function & Cost function，下面描述正确的是？
