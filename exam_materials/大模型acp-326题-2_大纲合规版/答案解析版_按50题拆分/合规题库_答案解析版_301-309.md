#### 第 301 题 [多选题]

某游戏平台上线了一个辅助聊天机器人，可帮助玩家完成游戏内交易、提交客服工单，以及获取实时游戏分析。有玩家发现机器人时常在简单客服请求时错误调用 analytics 工具。为改进这个多智能体系统的工具使用准确性，应该怎么做？

A. 删除 analytics 相关 Agent
B. 在 Agent 指令中进一步明确各 Agent 的功能边界
C. 改进 Planner Agent 的决策逻辑，精细化路由
D. 将游戏的所有代码库交给聊天机器人学习

**答案：`BC`**

**考点：** 多Agent及多模态应用 / Assistant API、智能体机制与工具调用

**详细解析：**

本题考查「Assistant API、智能体机制与工具调用」。关键词：Agent 负责规划/工具调用，多模态负责跨文本、图像、语音、视频。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `BC` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 不选：删除 analytics 相关 Agent
- B 正确：在 Agent 指令中进一步明确各 Agent 的功能边界
- C 正确：改进 Planner Agent 的决策逻辑，精细化路由
- D 不选：将游戏的所有代码库交给聊天机器人学习

**易错点分析：**

Agent 题要抓“任务拆解、工具描述、路由决策、函数调用、结果提交”，工具调用错常见原因是描述或参数 schema 不清。

**原题号：** 318

#### 第 302 题 [多选题]

你和同事在设计一个知识库答疑机器人，他提出了以下路线，哪两个是合理的？

A. 大模型能力很强，我们把所有知识文件的内容丢给大模型让它总结。
B. 我们可以通过人工与大模型对公司的知识文件生成很多问答对，再使用问答对微调大模型，使大模型了解我们公司的知识。
C. 相信大模型的能力，无需外部知识库支持，因为大模型已经预训练了海量数据。
D. 大模型最大输入长度有限，应该使用 RAG，先召回最相关的文本段，将文本段与输入问题进行拼接后输入大模型。

**答案：`BD`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

本题考查「RAG 核心流程、检索优化、文档解析/切片」。关键词：RAG = 外部知识库 + 检索相关片段 + 拼接上下文 + 生成答案。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `BD` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 不选：大模型能力很强，我们把所有知识文件的内容丢给大模型让它总结。
- B 正确：我们可以通过人工与大模型对公司的知识文件生成很多问答对，再使用问答对微调大模型，使大模型了解我们公司的知识。
- C 不选：相信大模型的能力，无需外部知识库支持，因为大模型已经预训练了海量数据。
- D 正确：大模型最大输入长度有限，应该使用 RAG，先召回最相关的文本段，将文本段与输入问题进行拼接后输入大模型。

**易错点分析：**

RAG 题优先判断问题发生在“解析、切片、向量化、召回、重排、生成”哪一层，不要把检索问题误当成单纯换大模型或调 temperature。
- A 是“换更强模型”的泛化思路，考试常考根因定位，不能默认选。
- C 偏训练/微调，若题目是知识更新或检索质量问题，通常不是低成本首选。

#### 第 303 题 [多选题]

qwen2.5-1.5b 模型训练过程中较大的 batch size 会带来哪些影响？

A. 帮助模型充分地学习训练集
B. 可能导致模型泛化性能下降
C. 加速训练过程
D. GPU 显存消耗更大

**答案：`BCD`**

**考点：** 大模型微调 / 微调概念、数据构建、训练参数与评测

**详细解析：**

本题考查「微调概念、数据构建、训练参数与评测」。关键词：用任务数据调整模型参数；适合稳定领域任务，不适合频繁更新事实知识。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `BCD` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 不选：帮助模型充分地学习训练集
- B 正确：可能导致模型泛化性能下降
- C 正确：加速训练过程
- D 正确：GPU 显存消耗更大

**易错点分析：**

微调题先抓训练闭环：数据质量、前向传播、损失、反向传播、梯度更新、验证集评估；不要把推理参数和训练参数混用。
- A 偏训练/微调，若题目是知识更新或检索质量问题，通常不是低成本首选。
