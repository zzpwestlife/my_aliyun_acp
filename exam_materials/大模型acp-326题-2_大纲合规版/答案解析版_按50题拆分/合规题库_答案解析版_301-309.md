#### 第 301 题 [多选题]

某游戏平台上线了一个辅助聊天机器人，可帮助玩家完成游戏内交易、提交客服工单，以及获取实时游戏分析。有玩家发现机器人时常在简单客服请求时错误调用 analytics 工具。为改进这个多智能体系统的工具使用准确性，应该怎么做？

A. 删除 analytics 相关 Agent

B. 在 Agent 指令中进一步明确各 Agent 的功能边界

C. 改进 Planner Agent 的决策逻辑，精细化路由

D. 将游戏的所有代码库交给聊天机器人学习

**答案：`BC`**

**考点：** 多Agent及多模态应用 / Assistant API、智能体机制与工具调用

**详细解析：**

本题考查「Assistant API、智能体机制与工具调用」。关键词：Agent 负责规划/工具调用，多模态负责跨文本、图像、语音、视频。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `BC` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 不选：删除 analytics 相关 Agent
  - 原因：删除工具不能解决路由判断错误的根因；问题出在 Planner 决策逻辑上，工具本身存在没有错。
- B 正确：在 Agent 指令中进一步明确各 Agent 的功能边界
  - 原因：工具/Agent 描述不清是路由错误的常见原因，明确功能边界能减少 Planner 的歧义判断。
- C 正确：改进 Planner Agent 的决策逻辑，精细化路由
  - 原因：Planner 是工具调用的决策核心，路由逻辑精细化直接解决错误调用问题。
- D 不选：将游戏的所有代码库交给聊天机器人学习
  - 原因：学习代码库与工具调用路由准确性无关，且成本极高，不是解决路由错误的有效手段。

**易错点分析：**

Agent 题要抓“任务拆解、工具描述、路由决策、函数调用、结果提交”，工具调用错常见原因是描述或参数 schema 不清。

**原题号：** 318

#### 第 302 题 [多选题]

你和同事在设计一个知识库答疑机器人，他提出了以下路线，哪两个是合理的？

A. 大模型能力很强，我们把所有知识文件的内容丢给大模型让它总结。

B. 我们可以通过人工与大模型对公司的知识文件生成很多问答对，再使用问答对微调大模型，使大模型了解我们公司的知识。

C. 相信大模型的能力，无需外部知识库支持，因为大模型已经预训练了海量数据。

D. 大模型最大输入长度有限，应该使用 RAG，先召回最相关的文本段，将文本段与输入问题进行拼接后输入大模型。

**答案：`BD`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

本题考查「RAG 核心流程、检索优化、文档解析/切片」。关键词：RAG = 外部知识库 + 检索相关片段 + 拼接上下文 + 生成答案。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `BD` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 不选：大模型能力很强，我们把所有知识文件的内容丢给大模型让它总结。
  - 原因：大模型有最大 context 长度限制，把所有文件全量输入不可行；且每次都全量输入成本极高，不是可持续方案。
- B 正确：我们可以通过人工与大模型对公司的知识文件生成很多问答对，再使用问答对微调大模型，使大模型了解我们公司的知识。
  - 原因：用问答对微调是让模型内化领域知识的有效手段，适合相对稳定的知识库场景。
- C 不选：相信大模型的能力，无需外部知识库支持，因为大模型已经预训练了海量数据。
  - 原因：预训练数据不包含公司私有知识，也存在知识截止日期，无法回答公司内部问题。
- D 正确：大模型最大输入长度有限，应该使用 RAG，先召回最相关的文本段，将文本段与输入问题进行拼接后输入大模型。
  - 原因：RAG 是解决 context 长度限制和私有知识更新的标准方案，按需检索效率更高。

**易错点分析：**

RAG 题优先判断问题发生在“解析、切片、向量化、召回、重排、生成”哪一层，不要把检索问题误当成单纯换大模型或调 temperature。
- A 是“换更强模型”的泛化思路，考试常考根因定位，不能默认选。
- C 偏训练/微调，若题目是知识更新或检索质量问题，通常不是低成本首选。

**原题号：** 319

#### 第 303 题 [多选题]

qwen2.5-1.5b 模型训练过程中较大的 batch size 会带来哪些影响？

A. 帮助模型充分地学习训练集

B. 可能导致模型泛化性能下降

C. 加速训练过程

D. GPU 显存消耗更大

**答案：`BCD`**

**考点：** 大模型微调 / 微调概念、数据构建、训练参数与评测

**详细解析：**

本题考查「微调概念、数据构建、训练参数与评测」。关键词：用任务数据调整模型参数；适合稳定领域任务，不适合频繁更新事实知识。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `BCD` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 不选：帮助模型充分地学习训练集
  - 原因：大 batch size 反而会减少参数更新次数（同等 epoch 下梯度更新步数更少），不利于充分学习；且更容易陷入 sharp minima，泛化性下降。
- B 正确：可能导致模型泛化性能下降
  - 原因：大 batch 训练容易收敛到 sharp minima（尖锐极小值），这类点对扰动敏感，泛化能力弱于 small batch 找到的 flat minima。
- C 正确：加速训练过程
  - 原因：大 batch 单次利用更多数据并行计算，GPU 利用率更高，每个 epoch 的时钟时间更短。
- D 正确：GPU 显存消耗更大
  - 原因：batch size 越大，单次前向+反向传播需要保存的中间激活值越多，显存占用线性增长。

**易错点分析：**

微调题先抓训练闭环：数据质量、前向传播、损失、反向传播、梯度更新、验证集评估；不要把推理参数和训练参数混用。
- A 偏训练/微调，若题目是知识更新或检索质量问题，通常不是低成本首选。

**原题号：** 320

#### 第 304 题 [多选题]

你发现检索到的切片里常常遗漏上下文，导致回答片段化。下列哪些是有效且合理的改进措施？

A. 改用 “句子窗口切片” 或 “滑动窗口检索”

B. 增大 chunk_overlap

C. 避免切片，将文档完整的拼接到提示词中

D. 适当调大召回文档切片数量（例如从 Top3 调整为 Top5）

**答案：`ABD`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

本题考查「RAG 核心流程、检索优化、文档解析/切片」。关键词：RAG = 外部知识库 + 检索相关片段 + 拼接上下文 + 生成答案。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `ABD` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 正确：改用 “句子窗口切片” 或 “滑动窗口检索”
  - 原因：句子窗口和滑动窗口在切分时保留相邻句子作为上下文，从切片策略层面直接解决上下文遗漏问题。
- B 正确：增大 chunk_overlap
  - 原因：chunk_overlap 让相邻切片之间有重叠内容，减少边界处上下文被截断的概率。
- C 不选：避免切片，将文档完整的拼接到提示词中
  - 原因：完整文档可能超出 LLM 最大 context 长度，且引入大量无关内容会稀释关键信息，成本也极高，不可持续。
- D 正确：适当调大召回文档切片数量（例如从 Top3 调整为 Top5）
  - 原因：召回更多切片能覆盖更大的上下文范围，降低漏掉关键片段的概率。

**易错点分析：**

RAG 题优先判断问题发生在“解析、切片、向量化、召回、重排、生成”哪一层，不要把检索问题误当成单纯换大模型或调 temperature。

#### 第 305 题 [多选题]

如果将 RAG 与开卷考试相类比，下列哪两项描述最符合它们的相同之处？

A. 都是通过引入外部知识来增强输出

B. 人的思维与大模型的权重参数，都会随着外部知识的输入而发生改变

C. 都需要提前对知识作出标记，来帮助检索

D. 都不需要对问题本身进行理解即可回答

**答案：`AC`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

本题考查「RAG 核心流程、检索优化、文档解析/切片」。关键词：RAG = 外部知识库 + 检索相关片段 + 拼接上下文 + 生成答案。

原解析要点：A 外部知识增强输出（RAG 检索知识，开卷考试查资料）；C 知识提前标记辅助检索；B RAG不改变模型权重，开卷不涉及权重类比；D 错误（均需理解问题）

选项判断：
- A 正确：都是通过引入外部知识来增强输出
  - 原因：RAG 检索外部知识库后拼入 prompt 生成答案；开卷考试查资料后作答——两者都是"用外部知识增强输出"的模式。
- B 不选：人的思维与大模型的权重参数，都会随着外部知识的输入而发生改变
  - 原因：RAG 推理时不修改模型权重，只是把检索结果注入 context；开卷考试也不改变人的长期记忆结构。类比不成立。
- C 正确：都需要提前对知识作出标记，来帮助检索
  - 原因：RAG 需要对文档分块、embedding 建索引；开卷考试需要书签、目录标注来快速定位内容——都需要提前整理知识以备检索。
- D 不选：都不需要对问题本身进行理解即可回答
  - 原因：RAG 需要理解 query 才能匹配相关文档；开卷考试也需要理解题目才能找到正确知识点。说法明显错误。

**易错点分析：**

RAG 题优先判断问题发生在“解析、切片、向量化、召回、重排、生成”哪一层，不要把检索问题误当成单纯换大模型或调 temperature。
- B 是“换更强模型”的泛化思路，考试常考根因定位，不能默认选。

#### 第 306 题 [多选题]

在开发一款基于大语言模型的会议纪要生成工具时，团队发现模型在处理会议记录时经常遗漏关键决议内容，导致生成的纪要不够准确和全面。以下哪两个选项能较低成本地解决这一问题？

A. 增加模型的训练数据量，特别是包含更多会议记录样本的数据集

B. 要求模型在输出纪要时使用特定格式（如紫色字体或表格形式）

C. 在提示词中明确角色定义，例如 “你是一名专业秘书，负责从会议记录中提取所有关键决议和行动项”

D. 要求模型先对会议记录中的行动项和决议进行逐句摘要标记，再生成最终的纪要

**答案：`CD`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

本题考查「RAG 核心流程、检索优化、文档解析/切片」。关键词：RAG = 外部知识库 + 检索相关片段 + 拼接上下文 + 生成答案。

原解析要点：选项通过提示词优化和流程设计，低成本解决模型遗漏关键内容的问题，而 A、B 分别因成本高或无关性被排除。

选项判断：
- A 不选：增加模型的训练数据量，特别是包含更多会议记录样本的数据集
  - 原因：重新训练成本极高，周期长；题目要求”较低成本”解决，训练数据扩充不符合这一约束。
- B 不选：要求模型在输出纪要时使用特定格式（如紫色字体或表格形式）
  - 原因：紫色字体是渲染层样式，LLM 输出纯文本，无法控制字体颜色；且改变格式不能解决”遗漏关键决议”的内容完整性问题。
- C 正确：在提示词中明确角色定义，例如 “你是一名专业秘书，负责从会议记录中提取所有关键决议和行动项”
  - 原因：明确角色和任务目标让模型聚焦于提取决议，直接解决”遗漏关键内容”的问题，成本仅是改 prompt。
- D 正确：要求模型先对会议记录中的行动项和决议进行逐句摘要标记，再生成最终的纪要
  - 原因：分步骤处理（先标记再汇总）是 Chain-of-Thought 思路，强制模型逐句过滤，大幅降低遗漏概率，低成本高效。

**易错点分析：**

RAG 题优先判断问题发生在“解析、切片、向量化、召回、重排、生成”哪一层，不要把检索问题误当成单纯换大模型或调 temperature。
- A 偏训练/微调，若题目是知识更新或检索质量问题，通常不是低成本首选。

#### 第 307 题 [多选题]

在评测机制里，你发现自动化评分一直偏高，但实际用户反馈一般。以下哪三项有可能提高评测的可信度？

A. 在构建测试集时，增加业务专家的打分环节

B. 调整自动化评测工程的提示词及阈值，让其更接近真实用户判断标注

C. 基于现有的评测集，增加自动化评测的频次

D. 根据线上用户问题类型的分布，修正测试样本集的问题分布

**答案：`ABD`**

**考点：** 大模型提示词工程 / 提示词框架、系统角色与任务设计

**详细解析：**

本题考查「提示词框架、系统角色与任务设计」。关键词：清晰角色、任务、上下文、输出格式、样例、约束。

解题思路：先定位题目所在环节，再看选项是否直接解决该环节的问题。正确答案 `ABD` 对应的是当前考点下的标准做法；未选项通常是把问题转移到无关环节、只改善体验不改善根因，或把训练/检索/生成/合规概念混淆。

选项判断：
- A 正确：在构建测试集时，增加业务专家的打分环节
  - 原因：业务专家标注引入真实业务判断标准，修正自动化评分与用户感知的偏差，提高评测数据质量。
- B 正确：调整自动化评测工程的提示词及阈值，让其更接近真实用户判断标注
  - 原因：自动评分偏高说明评测 prompt 或阈值与真实用户标准不对齐，校准二者直接解决偏差根因。
- C 不选：基于现有的评测集，增加自动化评测的频次
  - 原因：频次增加只是重复同样偏差的评测，不能提高可信度；根本问题是评测集和评分标准不准，不是次数不够。
- D 正确：根据线上用户问题类型的分布，修正测试样本集的问题分布
  - 原因：测试集分布与线上实际分布不一致会导致评分虚高；修正分布让评测更贴近真实使用场景。

**易错点分析：**

提示词题看是否明确了角色、任务、上下文、约束、输出格式和样例；含糊、只喊口号、没有格式约束的选项通常不优。

#### 第 308 题 [多选题]

在初步构建好大模型应用后，应该对它进行 “自动化评测”，如何理解自动化？

A. 自动化的评测指标由大模型制定

B. 自动化的评测集由大模型生成

C. 自动化指的是通过引入大模型来减少人为查验的工作方式

D. 自动化的前提是要有业务专家参与构建评测集

**答案：`CD`**

**考点：** 大模型应用开发 / 大模型适用场景与应用开发基础

**详细解析：**

本题考查「大模型适用场景与应用开发基础」。关键词：API 参数、消息角色、流式/批量、对话历史与适用边界。

原解析要点：A、B 错误 指标和评测集需人工主导，而非模型完全生成

选项判断：
- A 不选：自动化的评测指标由大模型制定
  - 原因：评测指标需要由业务专家和开发团队根据业务目标制定，让大模型自己制定指标会引入循环偏差（模型倾向给自己高分）。
- B 不选：自动化的评测集由大模型生成
  - 原因：评测集需要人工参与以确保质量和覆盖度，纯模型生成的测试集容易与训练分布重叠，无法检验真实弱点。
- C 正确：自动化指的是通过引入大模型来减少人为查验的工作方式

#### 第 309 题 [单选题]

公司新发了《远程办公规定》，你想让现有答疑机器人掌握这份规定的内容。基于已经搭建好的 RAG 应用，你应该怎么做？

A. 在提示词模板中增加远程办公规定的内容

B. 将新文档分块后更新向量数据库

C. 微调大模型以确保大模型能理解新规定的内容

D. 使用纯规则引擎匹配

**答案：`B`**

**考点：** 大模型检索增强 / RAG 核心流程、检索优化、文档解析/切片

**详细解析：**

本题考查「RAG 核心流程、检索优化、文档解析/切片」。关键词：RAG = 外部知识库 + 检索相关片段 + 拼接上下文 + 生成答案。

原解析要点：基于 RAG 架构，更新外部知识的正确方式是更新向量数据库（B 选项），这是 RAG 动态扩展知识的标准操作，低成本且高效

选项判断：
- A 不选：在提示词模板中增加远程办公规定的内容
  - 原因：把文档硬编码进 prompt 模板会超出 context 长度限制，且每次新增文档都要改代码，不可扩展；RAG 已有知识库，正确做法是更新知识库。
- B 正确：将新文档分块后更新向量数据库
  - 原因：RAG 架构下更新知识只需分块→embedding→写入向量库，不改代码、不重训模型，是最低成本的知识扩展方式。
- C 不选：微调大模型以确保大模型能理解新规定的内容
  - 原因：微调成本高、周期长，且频繁更新文件时无法持续微调；RAG 本就是为解决知识实时更新问题而设计的，无需微调。
- D 不选：使用纯规则引擎匹配
  - 原因：规则引擎只能做关键词精确匹配，无法理解语义；且已有 RAG 应用，改用规则引擎是降级退步，不是改进。

**易错点分析：**

RAG 题优先判断问题发生在“解析、切片、向量化、召回、重排、生成”哪一层，不要把检索问题误当成单纯换大模型或调 temperature。
- C 偏训练/微调，若题目是知识更新或检索质量问题，通常不是低成本首选。

**原题号：** 326
