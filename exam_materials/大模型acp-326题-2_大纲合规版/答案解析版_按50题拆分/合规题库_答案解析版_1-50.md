#### 第 1 题 [多选题]

以下哪些情况可能导致大模型检索不准确？

A. 文档中包含大量的图片信息，而当前的文档解析器无法有效提取和理解图片内容。
B. 使用了基于深度学习的语义搜索引擎，但模型训练数据不足，导致检索结果不准确。
C. 文档中包含复杂的表格结构，而简单的文本解析器无法理解表格单元格之间的关系。
D. 使用了复杂的文档解析器，导致解析速度慢，影响检索效率。
E. 索引构建过程中使用了错误的 embedding 模型，导致语义相似度计算错误。
F. 文档切片时，仅简单地按照字符长度进行分割，丢失了文档的语义信息。

**答案：`ABCEF`**

**考点：** 大模型检索增强 / RAG 文档解析、向量化、切片与召回准确性

**详细解析：**

**相关知识点：** RAG 准确性取决于资料是否被正确解析、切片是否保留语义、embedding 是否合适、召回排序是否可靠；速度慢属于效率问题，不等于答案不准。

**解题逻辑：** 本题不要只背答案，要先判断它考的是哪一层能力：RAG 题看解析、切片、向量化、召回、重排和生成；Prompt 题看角色、任务、上下文、约束、输出格式和样例；微调题看数据、训练循环、损失、梯度和验证集；Agent/API 题看消息、线程、运行、工具调用和返回结果。

**逐选项原因：**
- A 正确：文档中包含大量的图片信息，而当前的文档解析器无法有效提取和理解图片内容。
  - 原因：图片内容如果解析器无法抽取，就不会进入索引或 embedding，检索自然找不到相关语义。
- B 正确：使用了基于深度学习的语义搜索引擎，但模型训练数据不足，导致检索结果不准确。
  - 原因：语义检索模型训练不足会造成向量表达质量差，相似度排序不可靠。
- C 正确：文档中包含复杂的表格结构，而简单的文本解析器无法理解表格单元格之间的关系。
  - 原因：表格行列关系丢失后，单元格含义不完整，召回到的文本也可能答非所问。
- D 不选：使用了复杂的文档解析器，导致解析速度慢，影响检索效率。
  - 原因：解析器复杂主要增加耗时和资源消耗，题目问的是“检索不准确”，不是“检索慢”。
- E 正确：索引构建过程中使用了错误的 embedding 模型，导致语义相似度计算错误。
  - 原因：embedding 模型决定语义向量空间，模型选错会导致相似度计算偏离真实语义。
- F 正确：文档切片时，仅简单地按照字符长度进行分割，丢失了文档的语义信息。
  - 原因：纯按字符长度切分会截断句子或段落关系，造成上下文缺失。

**易错点分析：**

易把“慢”当成“不准”。考试里要区分准确率、召回率、延迟和吞吐。

#### 第 2 题 [多选题]

关于 Qwen-Max、CosyVoice 和 moviepy 的协同使用，以下哪些说法是正确的？

A. CosyVoice 可以将文本转换为音频。
B. CosyVoice 的主要功能是视频剪辑。
C. moviepy 可以直接将文本转换为视频。
D. Qwen-Max 主要用于生成视频字幕。
E. moviepy 可以用来获取音频文件的持续时间，用于生成字幕。
F. 使用这三个工具的典型流程是，Qwen-Max -> CosyVoice -> moviepy。

**答案：`AEF`**

**考点：** 多Agent及多模态应用 / 文本生成、语音合成与视频编辑工具协同

**详细解析：**

**相关知识点：** 典型多模态流水线是：大语言模型生成脚本/字幕文本，TTS 生成音频，视频编辑库负责合成与时间轴处理。

**解题逻辑：** 本题不要只背答案，要先判断它考的是哪一层能力：RAG 题看解析、切片、向量化、召回、重排和生成；Prompt 题看角色、任务、上下文、约束、输出格式和样例；微调题看数据、训练循环、损失、梯度和验证集；Agent/API 题看消息、线程、运行、工具调用和返回结果。

**逐选项原因：**
- A 正确：CosyVoice 可以将文本转换为音频。
  - 原因：CosyVoice 的核心能力是把文本转成语音，适合做旁白或语音助手声音。
- B 不选：CosyVoice 的主要功能是视频剪辑。
  - 原因：视频剪辑是 moviepy/ffmpeg 一类工具的职责，不是 CosyVoice 的主要功能。
- C 不选：moviepy 可以直接将文本转换为视频。
  - 原因：moviepy 能编辑和合成视频，但不能凭文本直接理解并生成完整视频内容。
- D 不选：Qwen-Max 主要用于生成视频字幕。
  - 原因：Qwen-Max 是通用文本生成大模型，可以生成字幕文本，但不是“主要用于字幕”的专用工具。
- E 正确：moviepy 可以用来获取音频文件的持续时间，用于生成字幕。
  - 原因：字幕时间轴需要知道音频时长，moviepy 可读取音频/视频时长辅助对齐。
- F 正确：使用这三个工具的典型流程是，Qwen-Max -> CosyVoice -> moviepy。
  - 原因：合理流程是先由 Qwen-Max 生成脚本，再用 CosyVoice 合成语音，最后由 moviepy 合成视频/字幕。

**易错点分析：**

不要把“生成字幕文本”“语音合成”“视频剪辑”混为一个工具能力。

#### 第 4 题 [多选题]

以下哪些因素会影响 RAG 系统的性能？

A. 硬件资源的限制
B. 文档的质量和数量
C. 向量数据库的类型和配置
D. 查询语句的表达方式
E. 所使用的预训练语言模型的大小和性能
F. 切片方法的选择

**答案：`ABCDEF`**

**考点：** 大模型检索增强 / RAG 系统端到端性能影响因素

**详细解析：**

**相关知识点：** RAG 是端到端系统，文档、向量库、查询、模型、切片、硬件都会影响最终效果或性能。

**解题逻辑：** 本题不要只背答案，要先判断它考的是哪一层能力：RAG 题看解析、切片、向量化、召回、重排和生成；Prompt 题看角色、任务、上下文、约束、输出格式和样例；微调题看数据、训练循环、损失、梯度和验证集；Agent/API 题看消息、线程、运行、工具调用和返回结果。

**逐选项原因：**
- A 正确：硬件资源的限制
  - 原因：硬件限制会影响索引构建、向量检索和生成延迟，进而影响系统性能。
- B 正确：文档的质量和数量
  - 原因：文档质量差、数量不足或噪声多，会直接影响召回内容。
- C 正确：向量数据库的类型和配置
  - 原因：向量数据库的索引类型、相似度算法和配置会影响检索速度与结果。
- D 正确：查询语句的表达方式
  - 原因：query 表达不清会导致向量匹配偏离真实意图。
- E 正确：所使用的预训练语言模型的大小和性能
  - 原因：生成模型能力影响最终答案质量，尤其是复杂推理和上下文整合。
- F 正确：切片方法的选择
  - 原因：切片大小、重叠和语义完整性会影响召回粒度和上下文质量。

**易错点分析：**

RAG 性能不是单点问题，题目问“哪些因素”时通常覆盖数据、检索、模型和资源全链路。

#### 第 5 题 [多选题]

通过 LlamaIndex 创建 RAG 应用，这段代码有哪些问题？

```python
query_engine = index.as_query_engine(
similarity_top_k=3,
streaming=True,
node_postprocessors=[
DepthScopeRank(top_n=8, model="gte-rerank"),
SimilarityPostprocessor(similarity_cutoff=1)]
)
response = query_engine.retrieve("需求分析使用的工具是什么？")
response.print_response_stream()
```

A. query_engine 不支持 retrieve
B. 重排选中的 chunk 数大于初步召回的 chunk 数
C. 相似度阈值设置过高
D. LlamaIndex 在 RAG 回答时不可以使用流式输出

**答案：`ABC`**

**考点：** 大模型检索增强 / LlamaIndex QueryEngine、rerank 与相似度阈值

**详细解析：**

**相关知识点：** LlamaIndex 中初召回数量、rerank 数量、相似度阈值和 query engine 调用方式必须匹配；流式输出本身不是错误。

**解题逻辑：** 本题不要只背答案，要先判断它考的是哪一层能力：RAG 题看解析、切片、向量化、召回、重排和生成；Prompt 题看角色、任务、上下文、约束、输出格式和样例；微调题看数据、训练循环、损失、梯度和验证集；Agent/API 题看消息、线程、运行、工具调用和返回结果。

**逐选项原因：**
- A 正确：query_engine 不支持 retrieve
  - 原因：QueryEngine 通常用于 query/stream_query 等生成式查询；直接 retrieve 属于 retriever 层能力，容易接口不匹配。
- B 正确：重排选中的 chunk 数大于初步召回的 chunk 数
  - 原因：初召回 similarity_top_k=3 只拿 3 个 chunk，rerank top_n=8 比候选还多，配置不合理。
- C 正确：相似度阈值设置过高
  - 原因：similarity_cutoff=1 过高，几乎要求完全相似，会过滤掉大量可用片段。
- D 不选：LlamaIndex 在 RAG 回答时不可以使用流式输出
  - 原因：LlamaIndex 支持流式输出，streaming=True 本身不是错误。

**易错点分析：**

代码题先看数量关系和接口层级：初召回 top_k、rerank top_n、cutoff、QueryEngine/Retriever。

#### 第 6 题 [多选题]

以下哪些描述符合基于语义的文档切片的理念？

A. 使用机器学习模型对文档进行语义分析，并根据语义边界进行切分。
B. 在 Chunk 中添加上下文信息，例如标题、父级列表项等。

**答案：`ABCF`**

**考点：** 大模型检索增强 / 语义切片与结构化文档处理

**详细解析：**

**相关知识点：** 语义切片追求语义完整和上下文可还原，不是机械按长度切，也不是把表格拆到失去行列语义。

**解题逻辑：** 本题不要只背答案，要先判断它考的是哪一层能力：RAG 题看解析、切片、向量化、召回、重排和生成；Prompt 题看角色、任务、上下文、约束、输出格式和样例；微调题看数据、训练循环、损失、梯度和验证集；Agent/API 题看消息、线程、运行、工具调用和返回结果。

**逐选项原因：**
- A 正确：使用机器学习模型对文档进行语义分析，并根据语义边界进行切分。
  - 原因：语义切片可以利用模型识别语义边界，比机械长度切分更能保持完整含义。
- B 正确：在 Chunk 中添加上下文信息，例如标题、父级列表项等。
  - 原因：标题、父级列表项等元数据能补足 chunk 的上下文。
- C 正确：将代码模块单独切出来，并标注其编程语言。
  - 原因：代码块有独立语义和语言属性，单独切分并标注有助于检索和理解。
- D 不选：将文档按照固定长度进行切分。
  - 原因：固定长度切分不关心语义边界，容易截断句子和逻辑单元。
- E 不选：将表格的每个单元格作为一个独立的 Chunk。
  - 原因：把每个单元格独立切开会丢失行列标题、单位和上下文。
- F 正确：根据文档的结构，例如标题、段落、列表等进行切分
  - 原因：按标题、段落、列表等结构切分更接近文档自然语义。

**易错点分析：**

语义切片不是越小越好，表格尤其不能拆到失去行列上下文。

#### 第 7 题 [多选题]

下列关于将 AgentModule 类设计为基类的原因，正确的有哪些？

```python
class AgentModule():
def __init__(self, name: str, description: str):
self.name = name
self.description = description
def __call__(self, query:str):
return self.query(query)
def query(self, query:str):
pass
```

A. 简化子类中的构造函数，减少重复代码
B. 由于 Python 不支持多重继承，因此需要这样设计
C. 允许每个子类定义自己的 query 方法，增加灵活性
D. 作为一个基类，可以方便地隐藏私有属性，保护数据
E. 提供统一的 call 方法以便通过相同的方式调用所有子类
F. 使得多个不同的代理模块可以共享相同的接口，实现代码复用

**答案：`ACEF`**

**考点：** 多Agent及多模态应用 / Agent 模块抽象、统一接口与工具封装

**详细解析：**

**相关知识点：** Agent 模块抽象的价值是统一调用方式和复用公共属性，让不同模块各自实现 query 逻辑。

**解题逻辑：** 本题不要只背答案，要先判断它考的是哪一层能力：RAG 题看解析、切片、向量化、召回、重排和生成；Prompt 题看角色、任务、上下文、约束、输出格式和样例；微调题看数据、训练循环、损失、梯度和验证集；Agent/API 题看消息、线程、运行、工具调用和返回结果。

**逐选项原因：**
- A 正确：简化子类中的构造函数，减少重复代码
  - 原因：基类统一保存 name、description，子类不用重复写相同初始化逻辑。
- B 不选：由于 Python 不支持多重继承，因此需要这样设计
  - 原因：Python 支持多重继承，该说法本身错误。
- C 正确：允许每个子类定义自己的 query 方法，增加灵活性
  - 原因：query 留给子类实现，允许不同 Agent 模块有不同处理逻辑。
- D 不选：作为一个基类，可以方便地隐藏私有属性，保护数据
  - 原因：基类主要解决接口和复用，不是 Python 私有属性保护机制。
- E 正确：提供统一的 call 方法以便通过相同的方式调用所有子类
  - 原因：__call__ 统一转发到 query，外部可以用相同方式调用不同模块。
- F 正确：使得多个不同的代理模块可以共享相同的接口，实现代码复用
  - 原因：共享接口能让 Planner 或上层流程用统一协议管理多个模块。

**易错点分析：**

别被 Python 语法细节带跑，ACP 真正考的是 Agent 统一接口和模块复用。

#### 第 8 题 [多选题]

关于大模型在文本处理中的局限性，以下哪些陈述是准确的？

#### 第 50 题 [单选题]

大语言模型微调相比重头训练的主要优势是什么？

- A 不选：训练时间更长

- B 正确：可以利用预训练学到的通用知识
  - 原因：预训练模型已有通用语言知识，微调是在此基础上适配任务。

  - 原因：微调通常比从零训练需要更少资源。
- D 不选：对特定任务的表现不如从头训练的模型
  - 原因：微调在特定任务上通常优于未适配或从零训练不足的模型。

微调的优势是复用预训练知识。
