# C3 考试大纲

本文档用于支撑 `C3` 模块题库升级，不仅整理《阿里云大模型 ACP 考试大纲》中与“构建 Agent 系统”相关的核心要求，也明确每类考点适合出的题型和难度层级，作为 `模拟题题目.md` 与 `模拟题答案解析.md` 的命题基线。

## 一、模块定位与权重

- 模块名称：`C3 构建 Agent 系统`
- 对应教程：`3_1` 到 `3_7`
- 对应考试大纲板块：多 Agent 及多模态应用
- 模块特征：更强调机制边界、模式选择、组合能力设计和评测闭环

按现有教程映射，`C3` 主要覆盖以下能力群组：

| 模块 | 对应教程 | 备考重点 |
| --- | --- | --- |
| Agent 运行机制与工具调用 | `3_1` | 工具调用边界、结构化输出、外部执行 |
| 工作流、规划与反思 | `3_2` | 固定工作流、自主规划、外部反馈 |
| 多 Agent 协作模式 | `3_3` | Leader-Worker、Blackboard、handoff |
| 记忆管理 | `3_4` | 无状态、截断、摘要、长期记忆 |
| Skill 设计与沉淀 | `3_5` | 从临时 Prompt 到可复用流程 |
| 评测驱动开发 | `3_6` | 端到端评测、白盒评测、持续迭代 |
| Qwen Code 实践 | `3_7` | Plan Mode、上下文管理、EPCC 工作流 |

## 三、命题难度分布建议

建议将 `C3` 题库组织为一套 `20` 题的全客观题标准样板：

- 单选题 `12`：基础 `5`、中档 `5`、拔高 `2`
- 多选题 `8`：基础 `3`、中档 `3`、拔高 `2`

对应命题重点如下：

| 难度 | 目标 | 典型题型 |
| --- | --- | --- |
| 基础 | 覆盖核心机制边界与基本运行逻辑 | 工具调用、无状态、Skill 基本概念 |
| 中档 | 引入任务约束和模式比较，提升区分度 | 工作流选择、协作模式、记忆策略 |
| 拔高 | 强调多能力组合与工程闭环 | 系统完整性、评测驱动迭代、能力组合判断 |

## 四、易错方向

- 把工具调用误解为模型直接执行工具，而不是“模型输出意图，外部程序执行”。
- 把固定工作流、自主规划、反思机制混为一谈，不会根据任务类型做选择。
- 只记多 Agent 模式名称，不会判断 Leader-Worker 与 Blackboard 的适用场景。
- 只知道记忆、Skill、评测这些能力存在，但不会把它们组合成可持续迭代系统。
- 把 Qwen Code 当作零散命令集合，而不是和 Agent 开发方法论关联起来理解。

## 五、从大纲到题目的落地原则

- 大纲先定义“考什么、考多深、适合出什么难度”。
- 题目再落实为“在什么场景下考、如何形成区分度、如何避免送分项”。
- 解析最后落实为“为什么对、为什么错、应该回到哪个知识点复习”。

这意味着 `C3` 不只是 Agent 概念归纳稿，也应当成为后续升级 `C4` 题目质量的参照样板。
