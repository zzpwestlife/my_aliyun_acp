# C3 模拟题题目

## 考试说明

- 本卷为 `C3` 质量升级扩展卷，共 `26` 题。
- 单选题 `16` 题，多选题 `10` 题。
- 难度结构：基础题 `8`、中档题 `8`、拔高题 `4`。
- 建议时长：`45` 到 `55` 分钟。
- 建议先独立作答，再对照 `模拟题答案解析.md` 复盘。

## 作答建议

- 先判断题目主要在考哪一层：工具、规划、协作、记忆、Skill、评测还是 Qwen Code。
- 中档题重点看清题干里的约束条件，再决定哪种机制或模式更合适。
- 拔高题不要只看“能力更多”的选项，而要判断是否形成完整的 Agent 工程闭环。

## 一、单选题

### 1. 【基础·3.1】某 Agent 在回答“帮我查明天杭州天气”时，先输出一个结构化对象表示要调用天气工具，再由外部程序真正执行查询。这说明工具调用的核心机制是？

- A. 模型直接执行了天气 API
- B. 模型生成调用意图，由外部程序执行工具
- C. 模型会自动遍历所有工具后再选择
- D. 工具调用本质上只是提高 `temperature`

### 2. 【基础·3.1】某团队希望降低工具调用失败率，要求模型输出稳定的参数结构，便于程序解析。以下哪项措施最直接？

- A. 用结构化约束限定输出字段
- B. 把系统提示词写得更长
- C. 只允许模型自然语言描述工具意图
- D. 强制每次都调用全部工具

### 3. 【中档·3.1】某平台接入天气、地图、日历等多个第三方能力，但不希望为每个服务都单独写一套接入协议。此时最需要的能力是？

- A. 通过微调让模型记住所有 API
- B. 使用统一的工具接入协议标准化能力暴露
- C. 为每个 API 写单独的长提示词模板
- D. 把所有外部工具改写成系统提示词

### 4. 【中档·3.2】某任务流程长期稳定，始终是“下载数据 -> 清洗 -> 汇总 -> 发邮件”。如果目标是降低不可控性并保证步骤顺序清晰，优先选择哪种组织方式？

- A. 完全依赖 Agent 自主探索路径
- B. 预设固定工作流或 Plan & Execute
- C. 让多个 Agent 共享白板自由讨论
- D. 只依赖长期记忆，不预设任何流程

### 5. 【中档·3.2】某 Agent 生成代码后静态检查通过，但真实运行报错。为了让 Agent 更快定位问题，最有效的下一步是？

- A. 让 Agent 仅根据已有代码再次自我反思
- B. 把真实报错信息和执行结果反馈给 Agent
- C. 立刻终止任务并更换模型
- D. 删除工具调用，改成纯文本推理

### 6. 【中档·3.3】某系统由一个总控 Agent 接收请求、拆分任务，再分发给检索 Agent、生成 Agent 和校验 Agent，最后总控 Agent 汇总结果。这最符合哪种协作模式？

- A. Blackboard 共创模式
- B. Leader-Worker 分层模式
- C. 单 Agent 多轮对话
- D. 长上下文记忆模式

### 7. 【中档·3.3】几个创意写作 Agent 围绕同一份“故事板”持续补充设定、改写情节、统一风格，没有固定的上下级，而是共享同一中间空间持续迭代。这更接近？

- A. Leader-Worker 分层调度
- B. Blackboard 共创协作
- C. 固定流水线工作流
- D. 单 Agent 外部工具调用

### 8. 【基础·3.4】开发者发现：如果新请求里不重新带入历史消息，模型就像“忘记了”上一轮对话。这最直接说明大模型具有？

- A. 有状态特性
- B. 无状态特性
- C. 自主记忆特性
- D. 离线缓存特性

### 9. 【中档·3.4】某客服 Agent 处理超长投诉时，早期对话里有用户身份和历史订单等关键背景。和直接丢弃早期历史相比，滚动摘要更合适的原因是？

- A. 完全不需要额外模型开销
- B. 压缩长度的同时尽量保留关键背景
- C. 一定比长期记忆检索更准确
- D. 能彻底替代所有记忆机制

### 10. 【基础·3.5】某团队每次做代码审查都要临时拼接很长 Prompt，且不同成员风格不一致。为了沉淀稳定能力，最适合的做法是？

- A. 把审查流程固化为可复用的 Skill
- B. 要求所有人把 Prompt 写得更长
- C. 只靠会话历史保留审查经验
- D. 把审查结论手工复制到系统提示词

### 11. 【中档·3.6】某 Agent 系统最终结果变差，团队怀疑问题出在中间组件，但仅看最终输出无法确定是检索模块还是生成模块导致。此时优先应该做什么？

- A. 只做端到端评测
- B. 针对内部模块设计白盒评测
- C. 同时替换全部模块后再观察
- D. 先凭经验判断问题来源

### 12. 【拔高·综合】某团队要构建一个可持续迭代的代码审查 Agent，要求它能读文件、调用搜索、遵循团队规范、跨会话记住项目约束，并持续评估质量。以下哪组能力组合最完整？

- A. 工具调用 + Skill + 持久记忆 + 评测闭环
- B. 长上下文 + 单次 Prompt + 会话历史 + 主观判断
- C. 外部脚本 + 人工复查 + 临时对话 + 一次性测试
- D. 大模型直答 + 无状态调用 + 无评测 + 硬编码规则

## 二、多选题

### 13. 【基础·3.1】某 Agent 接入工具时，模型偶尔输出了不符合预期的参数结构。为了提升工具调用可靠性，以下哪些做法更合理？

- A. 用结构化约束限制输出字段
- B. 对模型输出做程序校验
- C. 校验失败时把错误信息反馈给模型重试
- D. 完全依赖自然语言输出，不做解析

### 14. 【中档·3.2】关于工作流与反思机制，以下哪些说法更合理？

- A. 固定流程更适合步骤稳定、可重复的任务
- B. Plan & Execute 适合先拆解再按步骤推进
- C. 外部反馈比纯自我反思更适合发现运行时错误
- D. 所有任务都应完全交给 Agent 自主规划

### 15. 【基础·3.3】关于多 Agent 协作模式，以下哪些判断是正确的？

- A. Leader-Worker 更适合有明确拆解关系的任务
- B. Blackboard 更适合持续碰撞思路的开放任务
- C. handoff 可以作为协作组织方式之一
- D. 多 Agent 协作通常完全没有额外成本

### 16. 【中档·3.4】关于 Agent 的记忆管理，以下哪些做法是合理的？

- A. 用上下文截断控制短期上下文长度
- B. 用滚动摘要保留长对话中的关键信息
- C. 用向量检索为长期记忆提供召回入口
- D. 把所有对话不加筛选地永久写入长期记忆

### 17. 【中档·3.5】关于 Skill 的设计与演进，以下哪些说法更合理？

- A. Skill 有助于把重复使用的专业流程沉淀下来
- B. 高质量 Skill 需要考虑触发条件和流程组织
- C. Skill 纳入版本控制有利于团队共享和演进
- D. Skill 的目标是彻底消灭所有 Prompt

### 18. 【中档·3.6】关于评测驱动 Agent 开发，以下哪些说法更合理？

- A. 不能只凭“感觉还行”判断系统是否真的变好
- B. 端到端评测关注最终输出对用户是否有价值
- C. 白盒评测有助于定位中间组件的具体瓶颈
- D. 评测只需要在上线前做一次，后续无需持续跟踪

### 19. 【基础·3.7】关于 Qwen Code 的工作方式，以下哪些说法更合理？

- A. `Plan Mode` 适合先分析方案而不直接改代码
- B. `/compress` 适合在任务中压缩上下文
- C. `/clear` 适合在任务切换时清空上下文边界
- D. EPCC 的 `Explore` 阶段强调先快速改动，再回头理解现状

### 20. 【拔高·综合】某团队希望打造一个能持续优化的 Agent 开发流程，要求既能接外部能力、沉淀团队规范，又能记住长期约束并持续迭代。以下哪些设计更合理？

- A. 用工具调用或 MCP 连接文件、搜索等外部能力
- B. 把团队规范沉淀成 Skill，而不是每次临时粘贴 Prompt
- C. 用持久记忆载体保存跨会话项目约束
- D. 用端到端评测配合白盒评测持续优化系统

## 三、补充单选题

### 21. 【中档·3.1】某 Agent 接入“创建工单”工具后，偶尔会因为超时重试而把同一请求创建出两张重复工单。参数本身大多是正确的。若只能先补一项机制，最应该优先做什么？

- A. 提高 `temperature`，让模型生成更多不同写法
- B. 给写操作增加幂等键或请求去重机制，而不是只关注参数格式
- C. 把工具返回内容写得更长，提醒模型谨慎一点
- D. 完全取消自动重试，所有失败都直接终止

### 22. 【中档·3.2】以下哪类任务最不适合优先交给 Agent 做完全自主规划，而更适合预设固定工作流？

- A. 方案调研，目标是比较多种技术路线
- B. 报销审批流，步骤固定且错误执行会带来合规风险
- C. 创意文案改写，需要不断试不同风格
- D. 开放式头脑风暴，希望产生多个候选方向

### 23. 【中档·3.3】某系统包含多个专业 Agent：需求澄清 Agent 完成后，会把上下文和当前结论交给实现 Agent；实现 Agent 完成后，再移交给测试 Agent。整个过程中没有统一总控角色持续调度。这个模式更接近？

- A. Leader-Worker
- B. handoff
- C. Blackboard
- D. 长期记忆检索

### 24. 【中档·3.4】某团队把所有对话内容都写入长期记忆，结果记忆库越来越大，但召回出来的信息经常无关甚至过时。更合理的改进是？

- A. 继续把所有内容永久保存，只要向量库够大就能解决
- B. 只写入稳定、高价值、跨会话有复用意义的信息，并给记忆写入设置规则
- C. 放弃长期记忆，所有信息都塞进单次上下文
- D. 每次任务开始前清空全部记忆
