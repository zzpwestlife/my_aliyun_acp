## 一、单选题

### 1. 在调用大模型生成答复时，以下哪个参数最直接影响输出的随机性？
- A. `model`
- B. `temperature`
- C. `messages`
- D. `stream`

答案：B

解析：`temperature` 用于调节采样随机性，值越高通常输出越发散。`model` 决定模型版本，`messages` 承载上下文，`stream` 决定是否流式返回。

课程映射：`2_1_用大模型构建新人答疑机器人.ipynb`

考纲映射：大模型应用开发 -> 基本 API 参数如 `model`、`temperature`、`top_p`

### 2. 在构建支持连续追问的答疑机器人时，最关键的做法是：
- A. 只保留最后一次用户问题
- B. 每轮都随机切换模型
- C. 正确维护消息列表与对话历史
- D. 将所有问题改写成单选题

答案：C

解析：多轮对话依赖历史消息来维持上下文一致性，否则模型无法理解代词、省略和追问语义。其余选项都不能直接保证上下文连续性。

课程映射：`2_1_用大模型构建新人答疑机器人.ipynb`

考纲映射：大模型应用开发 -> 理解消息与对话历史

### 3. 在提示词工程中，系统角色提示词最核心的作用是：
- A. 为模型提供长期稳定的行为约束
- B. 替代全部用户输入
- C. 只用于控制输出长度
- D. 只在微调阶段生效

答案：A

解析：系统角色提示词用于定义身份、边界和输出风格，是较稳定的高优先级约束。它既不能替代用户输入，也不只和长度或微调相关。

课程映射：`2_3_优化提示词改善答疑机器人回答质量.ipynb`

考纲映射：大模型提示词工程 -> 理解系统角色提示词的作用

### 4. 以下哪项最符合课程中“利用大模型处理各类任务”的思路？
- A. 只能做闲聊，不能处理业务文本
- B. 只能在检索后输出原文，不能做理解与改写
- C. 可用于意图分类、文档审阅和自动修订等任务
- D. 必须先完成模型微调才能做任何任务

答案：C

解析：课程强调大模型不仅能问答，还能承担分类、审阅、修订等通用语言任务。A、B、D 都把模型能力限制得过窄。

课程映射：`2_3_优化提示词改善答疑机器人回答质量.ipynb`

考纲映射：大模型提示词工程 -> 利用大模型开发应用

### 5. 一个基础 RAG 流程中，较合理的顺序是：
- A. 文本切片 -> 文件解析 -> 段落召回 -> 重排序
- B. 文件解析 -> 文本切片 -> 段落召回 -> 重排序
- C. 重排序 -> 文件解析 -> 文本切片 -> 段落召回
- D. 段落召回 -> 文本切片 -> 文件解析 -> 重排序

答案：B

解析：RAG 通常先解析原始文档，再切片建立检索单元，随后完成召回与重排序。顺序颠倒会导致检索对象无法稳定构建。

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`

考纲映射：大模型检索增强 -> 文件解析、文本切片、段落召回、段落重排序

### 6. 关于 RAGAS 的描述，哪一项最准确？
- A. 它主要用于模型部署加速
- B. 它是 RAG 系统自动化评测的一类指标体系
- C. 它只用于图像分类
- D. 它用于替代知识库构建

答案：B

解析：RAGAS 是面向 RAG 应用评测的一套指标体系，用来帮助量化回答质量、检索支撑等方面表现。它既不是部署工具，也不负责知识库构建。

课程映射：`2_4_自动化评测答疑机器人的表现.ipynb`

考纲映射：大模型检索增强 -> 了解 RAGAS 指标体系

### 7. 当文档被切得较碎，导致回答缺少上下文时，以下哪种优化更有针对性？
- A. 提高 temperature
- B. 句子窗口检索
- C. 删除系统提示词
- D. 关闭重排序

答案：B

解析：句子窗口检索可以在命中局部片段后补回周边上下文，减少“召回到一句话但信息不完整”的问题。其余做法无法直接改善上下文缺失。

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 句子窗口检索等召回优化方法

### 8. 以下哪项更偏向“贴近实战的 RAG 优化方法”而不是基础调用参数设置？
- A. 调整 `top_p`
- B. 切换 `stream=True`
- C. 做标题改写与表格内容增强
- D. 修改 `messages` 的角色名

答案：C

解析：标题改写和表格增强是面向知识解析与召回效果的实战优化，属于 RAG 工程范畴。A、B、D 都更偏向调用层或对话组织层。

课程映射：`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> 标题改写优化、表格内容增强

## 二、多选题

### 9. 为了让问答机器人更稳定地处理多轮对话，以下哪些做法是合理的？
- A. 保留系统消息与关键历史轮次
- B. 对过长历史做裁剪或摘要
- C. 完全忽略 assistant 历史输出
- D. 让消息角色结构保持清晰

答案：A、B、D

解析：多轮对话需要结构化历史，同时要控制上下文长度。完全忽略 assistant 历史输出常会破坏对话连续性，因此 C 不合理。

### 13. 关于 RAG 自动化评测，以下说法哪些合理？
- A. 可以借助 RAGAS 指标体系评估系统表现
- B. 评测只看模型是否返回文本即可
- C. 评测有助于比较不同检索或提示策略
- D. 自动化评测能支持持续迭代优化

答案：A、C、D

解析：自动化评测的价值在于可量化、可比较、可持续迭代。只看“是否有文本输出”无法反映检索与答案质量，因此 B 错误。

## 三、场景题

### 14. 场景题：新人答疑机器人“能答但答不准”
某教育内容公司已经上线了一个新人答疑机器人。现在发现两个问题：一是员工换一种问法后，系统经常答不上来；二是即使答上来，也常只命中零碎片段，缺少完整上下文。

请给出一套优先级明确的改进方案，要求说明应先改哪里、为什么，并指出对应课程章节与 ACP 考纲落点。

参考答案：
1. 先补齐基础 RAG 流程，确保知识库解析、切片、召回、重排序链路完整，这是答不上来的第一层原因。
2. 再优化 Prompt 设计，包括系统角色、任务描述和输出模板，提升不同问法下的理解稳定性。
3. 对召回链路做进一步优化，如句子窗口检索、自动合并检索、标题改写和表格增强，解决“只命中零碎片段”的问题。
4. 最后引入自动化评测，用统一样本和指标比较优化前后的效果，避免只凭主观感受判断。

解析：该场景同时涉及 2.2、2.3、2.4、2.5。先保证基础链路可用，再做提示词与检索优化，最后用评测闭环验证，是更稳妥的工程路径。

课程映射：`2_2_扩展答疑机器人的知识范围.ipynb`、`2_3_优化提示词改善答疑机器人回答质量.ipynb`、`2_4_自动化评测答疑机器人的表现.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型检索增强 -> RAG 核心要素、RAG 优化、RAG 自动化评测；大模型提示词工程 -> 构建有效提示词

### 15. 场景题：如何用自动化评测指导迭代
你在对两版问答系统做对比：
- 版本 A：基础检索 + 朴素提示词
- 版本 B：加入标题改写、表格增强、重排序，并改进系统角色提示词

请说明你会如何设计一次自动化评测，至少回答以下三点：评什么、怎么比、比完后如何决策下一步。

参考答案：
1. 评什么：围绕代表性问答样本评估回答质量与检索支撑质量，重点关注 RAGAS 类指标以及问题覆盖面。
2. 怎么比：在同一批测试集上分别运行版本 A 和版本 B，比较回答正确性、检索相关性和整体稳定性，避免样本不一致造成偏差。
3. 如何决策：若版本 B 在主要指标上稳定优于版本 A，则保留优化方案；若部分问题仍表现较差，再回到切片策略、召回策略或提示词设计继续细化。

解析：自动化评测的关键不只是“打分”，而是让优化动作能被同口径比较，并形成持续迭代闭环。这也是 C2 从构建到优化再到评测的核心方法论。

课程映射：`2_3_优化提示词改善答疑机器人回答质量.ipynb`、`2_4_自动化评测答疑机器人的表现.ipynb`、`2_5_优化RAG应用提升问答准确度.ipynb`

考纲映射：大模型提示词工程 -> 系统角色提示词；大模型检索增强 -> RAGAS 指标体系、RAG 系统评测方法、RAG 优化方法
