# CourseReviewSkill

## 标准工作流程

1. 提取目录结构，先掌握全貌再逐章检查
2. 逐一执行所有代码块，验证可运行性（详见 [code-quality.md](code-quality.md)）
3. 检查技术概念的准确性（详见 [content-accuracy.md](content-accuracy.md)）
4. 评估难度梯度（知识点是否逐步递进，有无跳跃过大的地方）
5. 校对语言风格（详见 [style-guide.md](style-guide.md)）
6. 生成审核报告

## 反模式清单

- 不要修改代码中的变量名、API 版本号
- 不要跳过任何一项检查
- 不要在没有执行代码的情况下声称"代码可以正常运行"
- 不要给出"整体不错"之类的模糊结论

## 审查标准

每项检查必须给出：通过 / 不通过 / 需人工复核

## 输出格式

```
## 审查报告：{课程标题}

### 总评
- 结论：通过 / 不通过
- 发现问题：{N} 个（严重 {x} / 一般 {y} / 建议 {z}）

### 问题清单

#### [严重] {问题标题}
- 位置：{章节名} / Cell {N}
- 描述：{具体问题}
- 建议：{修改方案}
```
