# 代码可执行性检查项

在干净环境中逐一执行所有代码块，检查以下问题：

## 依赖完整性
- 是否有遗漏的依赖（如使用了 `openpyxl` 但未在环境准备中安装）
- 是否有未声明的系统依赖（如 Java、ffmpeg）

## 变量与函数
- 是否有未定义的变量或函数
- 变量是否在使用前赋值
- 函数调用的参数名是否与定义一致

## 文件与路径
- 引用的数据文件是否存在
- 路径写法是否跨平台兼容

## .ipynb 注意事项
- `.ipynb` 文件是 JSON 格式，代码在 `cells[*].source` 字段中
- 注意 Cell 执行顺序：上方 Cell 定义的变量，下方 Cell 才能使用
- 检查是否有需要重启内核才能复现的隐式状态依赖
